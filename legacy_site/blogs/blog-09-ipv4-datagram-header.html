Introduction and IPv4 Datagram Header

Networking January 02, 2026 Jobaer Hossain

Introduction

IPv4 (Internet Protocol version 4) is the most widely used version of the Internet Protocol. It defines how data is packetized, addressed, transmitted, routed, and received across networks.

IPv4 uses a 32-bit address scheme allowing for a total of 4,294,967,296 addresses. It operates at the Network Layer (Layer 3) of the OSI model and provides connectionless, best-effort delivery of datagrams.

IPv4 Datagram Header

The IPv4 packet is called a datagram. The header is usually 20 bytes long (without options) and contains information essential for routing and delivery.

Fields in IPv4 Header

Version (4 bits)

Indicates the IP version. For IPv4, this field is always 4.

Internet Header Length (IHL) (4 bits)

Specifies the length of the header in 32-bit words. Minimum value is 5 (20 bytes), maximum is 15 (60 bytes).

Type of Service (ToS) / DSCP + ECN (8 bits)

Used for Quality of Service (QoS). Originally for precedence, now divided into DSCP (6 bits) for packet classification and ECN (2 bits) for congestion control.

Total Length (16 bits)

Total size of the datagram (header + data) in bytes. Maximum size is 65,535 bytes.

Identification (16 bits)

Used to identify fragments of an original IP datagram during reassembly.

Flags (3 bits)

Control flags for fragmentation:

  • Bit 0: Reserved (must be 0)
  • Bit 1: Don't Fragment (DF)
  • Bit 2: More Fragments (MF)

Fragment Offset (13 bits)

Indicates the position of the fragment in the original datagram (in units of 8 bytes).

Time to Live (TTL) (8 bits)

Prevents datagrams from looping forever. Decremented by each router; packet discarded when TTL reaches 0.

Protocol (8 bits)

Indicates the next-level protocol (e.g., 1 = ICMP, 6 = TCP, 17 = UDP).

Header Checksum (16 bits)

Error-checking field for the header only (recalculated at each hop since TTL changes).

Source IP Address (32 bits)

IP address of the sender.

Destination IP Address (32 bits)

IP address of the intended recipient.

Options (variable, optional)

Rarely used; may include security, routing, timestamp, etc. Padded to 32-bit boundary.

Key Points

  • Minimum header size: 20 bytes
  • Maximum header size: 60 bytes (with options)
  • IPv4 is connectionless and unreliable (reliability handled by upper layers like TCP)

  • Fragmentation possible at routers if packet exceeds MTU

Summary

The IPv4 datagram header contains all necessary information for routing packets across interconnected networks. Despite its limitations (address space exhaustion, lack of built-in security), IPv4 remains the backbone of the Internet today.
