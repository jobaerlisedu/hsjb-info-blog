What is OSI Model? - 7 Layers Explained

Networking December 30, 2025 Jobaer Hossain

Introduction

The OSI (Open Systems Interconnection) Model is a set of rules that explains how different computer systems communicate over a network. Developed by the International Organization for Standardization (ISO), it consists of 7 layers, each with specific functions and responsibilities. This layered approach makes it easier for different devices and technologies to work together.

Note: The OSI Model provides a clear structure for data transmission and managing network issues. It is widely used as a reference to understand how network systems function.

The 7 Layers of the OSI Model

Layer 7: Application Layer

The topmost layer, closest to the end-user. It provides network services directly to applications.

  • Protocols: HTTP, FTP, SMTP, DNS, etc.
  • Functions: Network Virtual Terminal, File Transfer, Mail Services, Directory Services

Layer 6: Presentation Layer

Known as the Translation layer. Handles data formatting, encryption, and compression.

  • Functions: Translation (e.g., ASCII to EBCDIC), Encryption/Decryption, Compression
  • Protocols: TLS/SSL, JPEG, MPEG

Layer 5: Session Layer

Manages sessions between applications. Establishes, maintains, and terminates connections.

  • Functions: Session establishment/maintenance/termination, Synchronization, Dialog control
  • Protocols: NetBIOS, PPTP

Layer 4: Transport Layer

Provides end-to-end communication services for reliable data transfer.

  • Data Unit: Segments
  • Functions: Segmentation & Reassembly, Service Point Addressing, Flow/Error Control
  • Protocols: TCP, UDP

Layer 3: Network Layer

Handles routing and forwarding of packets between different networks.

  • Data Unit: Packets
  • Functions: Routing, Logical Addressing (IP)
  • Devices: Routers
  • Protocols: IP, ICMP

Layer 2: Data Link Layer

Provides node-to-node data transfer and error detection/correction.

  • Data Unit: Frames
  • Functions: Framing, Physical Addressing (MAC), Error/Flow/Access Control
  • Devices: Switches, Bridges
  • Protocols: Ethernet, PPP

Layer 1: Physical Layer

The lowest layer, responsible for physical transmission of bits.

  • Data Unit: Bits
  • Functions: Bit synchronization, Bit rate control, Physical topologies, Transmission mode
  • Devices: Hubs, Repeaters, Modems, Cables

How Data Flows in the OSI Model

Data travels down the 7 layers on the sender's side (encapsulation) and up the 7 layers on the receiver's side (decapsulation).

  1. Application → Data creation
  2. Presentation → Formatting & encryption
  3. Session → Connection management
  4. Transport → Segmentation & reliable delivery
  5. Network → Routing
  6. Data Link → Framing
  7. Physical → Bit transmission

Protocols in OSI Layers
