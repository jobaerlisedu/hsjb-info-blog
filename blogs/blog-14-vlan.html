Virtual LAN (VLAN): Detailed Guide

January 25, 2026 | Networking

Virtual LAN (VLAN) কী?

Virtual Local Area Network বা VLAN হলো একটি Layer 2 (Data Link Layer) Network-এর Logical বিভাজন, যা Device-গুলোকে তাদের Physical অবস্থান নির্বিশেষে Separate Separate Logical Network-এ Group করে। সাধারণ LAN-এর মতো Physical Topology-র ওপর নির্ভর না করে, VLAN Switch-এ IEEE 802.1Q Frame Tagging-এর মাধ্যমে Task করে।

একটি Physical Switch Network-কে Multiple Logical Broadcast Domain-এ বিভক্ত করে VLAN অপ্রয়োজনীয় Broadcast Traffic কমায়, Security বাড়ায় এবং Network Performance ও ব্যবস্থাপনার মান Advanced করে।

  • একই VLAN-এ: Device-গুলো সরাসরি একই Broadcast Domain-এর মধ্যে যোগাযোগ করতে পারে।
  • ভিন্ন VLAN-এ: যোগাযোগের জন্য একটি Router বা Layer 3 Switch-এর (SVI) মাধ্যমে Inter-VLAN Routing প্রয়োজন।
উল্লেখ্য: VLAN ছাড়া একটি সাধারণ LAN-এ Switch-এর সাথে সংযুক্ত সকল Device একটি মাত্র Broadcast Domain-এ থাকে। ফলে Broadcast Frame সব Port-এ ছড়িয়ে পড়ে, যা অপ্রয়োজনীয় Network Overhead এবং Security-র Risk তৈরি করে।

VLAN ID Range (Cisco Standard)

Cisco-এর Standard অনুযায়ী VLAN ID-গুলোকে নিচের ভাগে ভাগ করা হয়:

VLAN ID Range Type Description
VLAN 0 ও VLAN 4095 Reserved IEEE 802.1Q দ্বারা সংরক্ষিত। Protocol Operation-এর জন্য ব্যবহৃত; Configure করা যায় না।
VLAN 1 Default VLAN Cisco Switch-এর Default VLAN। সব Port Default-এ এখানে থাকে। STP, CDP, VTP-র জন্য ব্যবহৃত। মুছে ফেলা যায় না।
VLAN 2–1001 Normal VLAN সাধারণ VLAN Configuration-এর জন্য ব্যবহৃত। সম্পূর্ণ Configurable এবং Switch-এর VLAN Database-এ সংরক্ষিত।
VLAN 1002–1005 Reserved (Legacy) FDDI এবং Token Ring-এর মতো পুরনো প্রযুক্তির জন্য সংরক্ষিত। মুছে ফেলা যায় না।
VLAN 1006–4094 Extended VLAN বড় Network-এর জন্য Design করা। Running Configuration-এ সংরক্ষিত এবং VTP Transparent Mode প্রয়োজন।

VLAN Configuration Example

নিচে Cisco Switch-এ VLAN তৈরি এবং Port-এ Assign করার একটি বাস্তব Example দেওয়া হলো:

Step 1: VLAN তৈরি করুন

Switch(config)# vlan 2
Switch(config-vlan)# name Accounts
Switch(config)# vlan 3
Switch(config-vlan)# name HR

Step 2: Switch Port-এ VLAN Assign করুন

Switch(config)# interface fa0/0
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 2

Switch(config)# interface fa0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 3
এখানে Fa0/0 Port-টি VLAN 2 (Accounts)-এ এবং Fa0/1 Port-টি VLAN 3 (HR)-এ Assign করা হয়েছে। যেহেতু এই দুটি Separate Broadcast Domain, তাই এদের মধ্যে যোগাযোগের জন্য Inter-VLAN Routing প্রয়োজন।

VLAN Link Type-সমূহ

Access Link

VLAN সম্পর্কে অজ্ঞ End-device (যেমন PC) থেকে VLAN-aware Switch-এ সংযোগ। Frame-গুলো Untagged হয় এবং একটি মাত্র VLAN-এর অন্তর্গত থাকে।

Trunk Link

দুটি VLAN-aware Device-এর মধ্যে (যেমন Switch-to-Switch বা Switch-to-Router) সংযোগ। IEEE 802.1Q Tagging ব্যবহার করে Multiple VLAN-এর Traffic বহন করে।

Hybrid Link

একই Link-এ Tagged এবং Untagged উভয় Frame সমর্থন করে। এটি সাধারণত Vendor-specific এবং IEEE 802.1Q Standard-এর অংশ নয়।

VLAN-এর Feature সমূহ (Features)

  • VLAN Tagging (IEEE 802.1Q): Ethernet Frame-এ একটি 4-byte VLAN Tag যোগ করা হয় যা Frame-টি কোন VLAN-এর তা Symbol িত করে।
  • VLAN Membership: Device-গুলোকে Switch Port, MAC Address বা Protocol Type-এর ভিত্তিতে VLAN-এ Assign করা যায়।
  • VLAN Trunking: একটি Physical Link-এ Multiple VLAN-এর Traffic Share করার Advantage দেয়।
  • Dynamic VLANs: পূর্ব Fixed Policy বা Authentication প্রক্রিয়ার ওপর ভিত্তি করে স্বয়ংক্রিয়ভাবে Device-কে VLAN-এ Assign করা হয়।

VLAN-এর বাস্তব প্রয়োগসমূহ

VoIP:

Dedicated Voice VLAN Quality of Service (QoS) নিশ্চিত করে Voice Traffic-কে অগ্রাধিকার দেয়।

Video Conferencing:

VLAN প্রাধান্য Latency এবং Jitter কমিয়ে কল কোয়ালিটি Advanced করে।

Cloud & Data Center:

Security ও Traffic পৃথকীকরণের জন্য ভাড়াটে Workload Logically বিচ্ছিন্ন রাখা হয়।

IoT Network:

IoT Device-গুলো Separate VLAN-এ রাখা Security Risk কমায়।

Gaming Network:

VLAN Gaming Traffic-কে অগ্রাধিকার দিয়ে Lag ও Packet Loss কমায়।

Remote Access:

Secure VLAN VPN ব্যবহারকারী ও বাহ্যিক সংযোগ Operation করে Network বিচ্ছিন্ন রাখে।

VLAN-এর Advantage ও অ Advantage

Pros

  • Advanced Security: সংবেদনশীল Traffic Separate VLAN-এ বিচ্ছিন্ন থাকে।
  • Better Performance: Broadcast Traffic সীমিত করে Network Load কমায়।
  • সহজ ব্যবস্থাপনা: বিভাগ অনুযায়ী (HR, Finance) Logical Grouping করা সহজ।
  • Flexible তা: Physical Cable Change ছাড়াই Device VLAN Change করা যায়।
  • সাশ্রয়ী: অতিরিক্ত Physical Network ও Routing Hardware-এর প্রয়োজনীয়তা কমায়।
  • Scalability: বড় Network-কে ছোট ছোট Operation যোগ্য অংশে ভাগ করা যায়।

Cons

  • Configuration Complexity: সঠিক Plan ও Configuration প্রয়োজন।
  • VLAN ID সীমাবদ্ধতা: Standard VLAN ID সীমিত, যা অনেক বড় Network-এ সমস্যা হতে পারে।
  • Security Risk: ভুল Configuration VLAN Hopping Attack-এর সুযোগ তৈরি করতে পারে।
  • Interoperability সমস্যা: Non-standard বা পুরনো Device-এর সাথে সামঞ্জস্য সমস্যা দেখা দিতে পারে।
  • Troubleshooting কঠিন: বিচ্ছিন্ন Traffic প্রবাহ Error নির্ণয়কে জটিল করে তোলে।

Summary

VLAN Modern Network Design-এর একটি অপরিহার্য Component। এটি একটি Physical Infrastructure Multiple Logical Network তৈরি র Advantage দেয়, Security বাড়ায় এবং Performance Advanced করে। সঠিকভাবে Configure করা হলে VLAN যেকোনো আকারের Network-এর দক্ষতা ও Security নিশ্চিত করতে সক্ষম।
