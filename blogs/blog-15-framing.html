Framing in Data Link Layer: Detailed Guide

February ০১, 2026 | Networking

Framing What is?

একটি Frame হলো Data Link Layer-এ Data Transmission-এর মৌলিক Single। এটি Bit-এর একটি সংগঠিত গোষ্ঠী যেখানে মূল Information ছাড়াও Address ও Error Detection-এর মতো Control Information থাকে।

Preamble / Flag
Dest. Address
Src. Address
Length / Type
Data (Payload)
FCS / CRC
End Flag
  • Frame-এ Address ও Error Detection-এর মতো Control Information থাকে।
  • এটি Data Link Layer (DLL)-এ সম্পাদিত হয়।
  • Bit stream-কে ছোট ও অর্থবহ Singleে বিভক্ত করে।
  • Sender ও Receiver-এর মধ্যে Synchronization নিশ্চিত করে।
  • Receiver-এর কাছে Reliable Transmission ও সঠিক Processing নিশ্চিত করে।
উল্লেখ্য: Framing Time Division Multiplexing (TDM)-এর মতো Techniqueেও অত্যন্ত গুরুত্বপূর্ণ, যেখানে Data নির্দিষ্ট Time Slot-এ পাঠানো হয়। Ethernet, Token Ring এবং Frame Relay-এর মতো প্রযুক্তি সুনির্দিষ্ট Frame Format ব্যবহার করে।

Framing-এর Purpose (Purpose of Framing)

  • Frame Boundary Determination: স্পষ্ট সীমানা Determination করে যাতে Receiver প্রতিটি Frame সঠিকভাবে Symbolিত করতে পারে।
  • Address সংযোজন: Source ও Destination Address বহন করে সঠিক Point-to-Point যোগাযোগ নিশ্চিত করে।
  • Error Detection & Correction: Data Integrity বজায় রাখতে সহায়তা করে।
  • Reliable Transmission: সুসংগঠিত ও Reliable Data পাঠানো নিশ্চিত করে।
মনে রাখুন: Framing ও Reassembly প্রক্রিয়া সম্পূর্ণভাবে Data Link Layer দ্বারা পরিচালিত হয় এবং ব্যবহারকারীর কাছে এটি সম্পূর্ণ Transparent থাকে।

Framing-এর Classification (Types of Framing)

1. Fixed-Size Framing

এই Methodতে প্রতিটি Frame-এর একটি পূর্বFixed স্থায়ী Size থাকে।

  • Frame-এর Lengthই সীমানা Determination করে বলে Separate Delimiter-এর প্রয়োজন হয় না।
  • অAdvantage: যখন Data Frame-এর Sizeের চেয়ে ছোট হয়, তখন Internal Fragmentation হয়, যা Bandwidth-এর অপচয় করে।
  • সমাধান: অব্যবহৃত স্থান পূরণ করতে Padding ব্যবহার করা হয়।

2. Variable-Size Framing

এই Methodতে Frame-এর শুরু ও শেষ Symbolিত করার জন্য স্পষ্ট সীমানা Instructionক ব্যবহার করা হয়। দুটি ুপায়ে এটি করা হয়:

Length Field:

Frame-এর মধ্যে একটি Field থাকে যা Frame-এর Size Instruction করে (IEEE 802.3 Ethernet-এ ব্যবহৃত)। এই Field নষ্ট হলে Receiver Frame-টি সঠিকভাবে বুঝতে পারে না।

End Delimiter:

একটি অনন্য Pattern ব্যবহার করে Frame-এর শেষ Symbolিত করা হয় (Token Ring-এ ব্যবহৃত)। তবে এই Pattern Data-র ভেতরেও থাকতে পারে, যা সমস্যা Creation করে।

এই সমস্যা সমাধানের জন্য Stuffing Technique ব্যবহার করা হয়:

(ক) Character/Byte Stuffing

  • Character-ভিত্তিক Data-র Fieldে ব্যবহৃত হয়।
  • যদি Data-র মধ্যে বিশেষ Delimiter Character (যেমন $) থাকে, তাহলে তার আগে একটি Escape Character (যেমন \O) যোগ করা হয়।
Example:
ED (End Delimiter) = $
Original Data: Hello $ World
After Stuffing: Hello \O$ World

Disadvantage: Processing overhead High এবং Modern সিস্টেMay এটি পুরনো।

(খ) Bit Stuffing

  • End Delimiter (ED) হলো একটি Bit Sequence, যেমন 01111।
  • যখনই Data-য় এই Pattern দেখা যায়, বিভ্রান্তি এড়াতে একটি অতিরিক্ত 0 Bit প্রবেশ করানো হয়।
Example ১:
Original Data: 011100011110
ED Pattern: 0111
After Bit Stuffing: 011010001101100

Example ২:
Original Data: 110001001
ED Pattern: 1000
After Bit Stuffing: 11001010011

Framing-এর সমস্যাসমূহ (Problems in Framing)

শুরু ও শেষ শনাক্তকরণ:

Receiver অবশ্যই delimiter বা flag ব্যবহার করে Frame সীমানা সঠিকভাবে Symbolিত করতে হবে।

Synchronization:

High-Speedর Network-এ Sender ও Receiver-কে Frame-এর Timing-এ একমত থাকতে হবে।

Error Handling:

Noise Data বা Delimiter নষ্ট করতে পারে। CRC বা Checksum-এর মতো Method ব্যবহার করা হয়।

Overhead:

Header ও Trailer Control Information বহন করে কিন্তু ব্যবহারযোগ্য Bandwidth কমিয়ে দেয়।

Efficiency:

Overhead ও Processing বিলম্ব কমিয়ে Payload সর্বাধিক করাই লক্ষ্য।

Compatibility:

ভিন্ন Device ভিন্ন Framing Method ব্যবহার করতে পারে, যা Data-র ভুল ব্যাখ্যা করার Reason হতে পারে।

Summary

Framing হলো Data Link Layer-এর একটি মৌলিক প্রক্রিয়া যা Bit-এর অবিচ্ছিন্ন প্রবাহকে অর্থবহ Singleে বিভক্ত করে। Fixed-size Framing সহজ হলেও Padding-এর প্রয়োজন হয়, আর Variable-size Framing Flexible হলেও Stuffing Techniqueের মাধ্যMay Delimiter-এর সমস্যা সামলাতে হয়। সঠিক Framing ছাড়া Reliable Network যোগাযোগ সম্ভব নয়।
