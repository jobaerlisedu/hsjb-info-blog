Flow Control: Data Link Layer

February 08, 2026 | Networking

Flow Control What is?

Flow Control হলো OSI Model-এর Data Link Layer-এর একটি প্রক্রিয়া যা Sender ও Receiver মধ্যে Data Transmission-এর Speed নিয়ন্ত্রণ করে। এটি নিশ্চিত করে যে Sender এমন Speed তে Data পাঠায় যা Receiver সামলাতে পারে — Buffer Overflow এবং Data হারানো রোধ করে।

  • Sender ও Receiver-এর মধ্যে Data-র প্রবাহ নিয়ন্ত্রণ করে।
  • Transmission Speed ও Processing স Capacity র পার্থক্য সামলায়।
  • Buffer Overflow ও Data হারানো রোধ করে।
  • Device-গুলির মধ্যে একটি Speed-matching mechanism হিসেবে Task করে।
  • ACK পাওয়ার আগে কতটি Frame পাঠাবে তা নিয়ন্ত্রণ করে।
  • Data Transmission Rate ও Memory Capacity-র মধ্যে ভারসাম্য রাখে।

Flow Control-এর Method সমূহ (Approaches)

1. Feedback-Based Flow Control

এখানে Receiver Data পাওয়ার Capacity সম্পর্কে Sender-কে সরাসরি Feedback পাঠায় এবং Sender সেই অনুযায়ী Speed সামঞ্জস্য করে।

  • Sender ACK বা Window Update পাওয়ার পরেই Data পাঠায়।
  • Data পাঠানোর হার Receiver-এর বর্তমান অবস্থা বা Network Environment অনুযায়ী Speed শীলভাবে পরিবর্তিত হয়।
  • TCP-এর মতো Protocol-এ ব্যাপকভাবে ব্যবহৃত হয়।

Computer Architecture-এ ব্যবহৃত Technique:

Ready-Signal Handshaking:

ভিন্ন Speed র Component-এর মধ্যে ব্যবহৃত হয়। Receiver "ready" Signal পাঠালে তবেই Data Transfer শুরু হয়।

Credit-Based Flow Control:
  • Fibre Channel-এর মতো High-speed Protocol-এ ব্যবহৃত।
  • Receiver Buffer Space-এর সমপরিমাণ Credit দেয়; Sender প্রতিটি Credit-এর বিপরীতে একটি Frame পাঠাতে পারে।
  • Receiver Data Process করার সাথে সাথে Credit পুনরুদ্ধার হয়।
  • সাধারণ Stop-and-Wait Flow Control-এর চেয়ে High দক্ষ।

2. Rate-Based Flow Control

Sender Receiver-এর ক্রমাগত Feedback ছাড়াই একটি নির্দিষ্ট বা পূর্ব Fixed হারে Data পাঠায়।

  • Sender স্বাধীনভাবে Transmission Rate নিয়ন্ত্রণ করে।
  • ACK বা Window Update-এর ওপর নির্ভর করে না।
  • Streaming, Multimedia ও Hardware System-এ উপযুক্ত।

Computer Architecture-এ ব্যবহৃত Technique:

Leaky Bucket Algorithm:

Input Traffic-কে একটি নির্দিষ্ট হারে Output করে, Bursty Traffic-কে মসৃণ করে।

Token Bucket Algorithm:

নির্দিষ্ট হারে Token Creation হয়; Token থাকলেই Data পাঠানো যায়। বড় Bucket ছোট Burst Support করে।

Traffic Shaping & Policing:

Shaping অতিরিক্ত Packet বিলম্বিত করে, Policing অতিরিক্ত Packet বাদ দেয়।

Flow Control-এর Technique সমূহ (Techniques)

1. Stop-and-Wait Flow Control

একটি সহজ Feedback-Based Method যেখানে Sender একটি Frame পাঠিয়ে ACK পাওয়ার জন্য অপেক্ষা করে, তারপর পরের Frame পাঠায়।

  • সহজ ও বাস্তবায়নযোগ্য।
  • Reliable Data Transmission নিশ্চিত করে।
  • অ Advantage: High-Latency বা High-Bandwidth Network-এ খারাপ Link ব্যবহার — অপেক্ষার Reason ে Line খালি পড়ে থাকে।

2. Sliding Window Flow Control

Sender ACK পাওয়ার আগেই Multiple Frame পাঠাতে পারে। Receiver-এর ACK পাওয়ার সাথে সাথে Window সামনে এগিয়ে (slide) যায়, যা Pipelined Transmission সম্ভব করে।

  • Pipelined Transmission-এর মাধ্যমে Bandwidth ব্যবহার Advanced করে।
  • একই সাথে Multiple Frame Transit-এ থাকতে পারে।
  • Window ও Buffer Management-এর জন্য অতিরিক্ত Memory ও Complexity প্রয়োজন।

Flow Control-এর গুরুত্ব

Data হারানো রোধ:

Receiver-এর Buffer Overflow এড়াতে Transmission Rate নিয়ন্ত্রণ করে।

Throughput Advanced ি:

কোনো Device-কে Overload না করে দক্ষতার সাথে Network Resource ব্যবহার নিশ্চিত করে।

System Stability:

অতিরিক্ত Data Input-এর Reason ে Hardware ও Software স্তব্ধ হওয়া বা ব্যর্থ হওয়া রোধ করে।

Fairness নিশ্চিতকরণ:

Switch ও Router-এর মতো Shared Environment ে Multiple সংযোগে সুষম Data Transmission নিশ্চিত করে।

Summary

Flow Control Data Link Layer-এর একটি অবিচ্ছেদ্য অংশ। Stop-and-Wait Method সহজ কিন্তু কম দক্ষ, আর Sliding Window Method High দক্ষ ও Modern Network-এ ব্যাপকভাবে ব্যবহৃত হয়। Feedback-based ও Rate-based দুটি Method ই বিভিন্ন Situation তে Network-এর কার্যকারিতা নিশ্চিত করে।
