Client-Server Model & Architecture

March 15, 2026 System Design | Networking

তথ্যপ্রযুক্তির বর্তমান যুগে ইন্টারনেট বা নেটওয়ার্কে আমরা যে সুবিধাই গ্রহণ করি না কেন—হোক তা ওয়েবসাইট ব্রাউজিং, ইমেইল পাঠানো কিংবা অনলাইন ব্যাংকিং—তার বেশিরভাগই একটি নির্দিষ্ট নিয়মের ওপর ভিত্তি করে চলে। এই নিয়ম বা কাঠামোর নামই হলো Client-Server Model। এটি এমন একটি নেটওয়ার্ক আর্কিটেকচার যেখানে 'ক্লায়েন্ট' রিসোর্সের জন্য রিকোয়েস্ট করে এবং 'সার্ভার' সেই রিকোয়েস্ট অনুযায়ী ডেটা বা সার্ভিস প্রদান করে।

মূল উপাদানসমূহ

  • Client (ক্লায়েন্ট): যে ডিভাইস বা প্রোগ্রাম সার্ভিস বা ডেটার জন্য রিকোয়েস্ট করে। (যেমন: আপনার ল্যাপটপের Google Chrome ব্রাউজার)।
  • Server (সার্ভার): এমন একটি শক্তিশালী সিস্টেম যা রিসোর্স বা ডেটা সংরক্ষণ করে এবং ক্লায়েন্টের রিকোয়েস্ট অনুযায়ী তা প্রদান করে।

এটি কীভাবে কাজ করে?

Client-Server মডেলটি মূলত Request–Response Mechanism-এর ওপর নির্ভর করে কাজ করে। এর কাজের ধাপগুলো খুবই সহজ:

  1. Request: প্রথমে ক্লায়েন্ট (যেমন ব্রাউজার) সার্ভারের কাছে কোনো নির্দিষ্ট সার্ভিস বা ডেটার জন্য রিকোয়েস্ট পাঠায়।
  2. Processing: সার্ভার সেই রিকোয়েস্টটি গ্রহণ করে এবং যাচাই-বাছাই (Processing/Authentication) করে। সার্ভার একসাথে শত শত ক্লায়েন্টের রিকোয়েস্ট হ্যান্ডেল করতে পারে।
  3. Response: প্রসেসিং শেষে সার্ভার কাঙ্ক্ষিত ফলাফল বা ডেটা রেসপন্স হিসেবে ক্লায়েন্টের কাছে ফেরত পাঠায়।
Examples of Clients

Web Browsers (Chrome, Firefox), Email Apps (Gmail, Outlook), Mobile Banking Apps.

Examples of Servers

Web Servers (Apache, Nginx), Email Servers, Database Servers (MySQL, Oracle).

C++ এ Client-Server Communication কীভাবে কাজ করে?

সি ল্যাঙ্গুয়েজ এবং C++ এ ক্লায়েন্ট ও সার্ভারের মধ্যে নেটওয়ার্কের মাধ্যমে যোগাযোগ স্থাপন করতে Sockets ব্যবহার করা হয়। নিচে একটি ব্যাসিক ক্লায়েন্ট এবং সার্ভারের কোড দেওয়া হলো যা 8080 পোর্টে একে অপরের সাথে কানেক্ট করে।

Server Code (server.cpp)

server.cpp C++ 
#include <iostream>
#include <cstring>
#include <unistd.h>
#include <netinet/in.h>

int main() {
    int server_fd, new_socket;
    struct sockaddr_in address;
    int addrlen = sizeof(address);
    char buffer[1024] = {0};

    // Create socket
    server_fd = socket(AF_INET, SOCK_STREAM, 0);

    // Setup address
    address.sin_family = AF_INET;
    address.sin_addr.s_addr = INADDR_ANY;
    address.sin_port = htons(8080);

    // Bind socket
    bind(server_fd, (struct sockaddr*)&address, sizeof(address));

    // Start listening
    listen(server_fd, 3);
    std::cout << "Server waiting for connection...\n";

    // Accept a client connection
    new_socket = accept(server_fd, (struct sockaddr*)&address, (socklen_t*)&addrlen);

    // Read message from client
    read(new_socket, buffer, 1024);
    std::cout << "Client says: " << buffer << std::endl;

    // Send reply
    const char* reply = "Hello from server!";
    send(new_socket, reply, strlen(reply), 0);

    close(new_socket);
    close(server_fd);
    return 0;
}

Client Code (client.cpp)

client.cpp C++ 
#include <iostream>
#include <cstring>
#include <unistd.h>
#include <arpa/inet.h>

int main() {
    int sock = 0;
    struct sockaddr_in serv_addr;
    char buffer[1024] = {0};

    // Create socket
    sock = socket(AF_INET, SOCK_STREAM, 0);

    // Setup Server Address
    serv_addr.sin_family = AF_INET;
    serv_addr.sin_port = htons(8080);
    inet_pton(AF_INET, "127.0.0.1", &serv_addr.sin_addr);

    // Connect to server
    connect(sock, (struct sockaddr*)&serv_addr, sizeof(serv_addr));

    // Send message
    const char* hello = "Hello from client!";
    send(sock, hello, strlen(hello), 0);

    // Receive reply
    read(sock, buffer, 1024);
    std::cout << "Server says: " << buffer << std::endl;

    close(sock);
    return 0;
}

আর্কিটেকচারের ধরন (Types of Architecture)

প্রেজেন্টেশন, লজিক এবং ডেটা লেয়ারগুলো কীভাবে সাজানো তার ওপর ভিত্তি করে Client-Server আর্কিটেকচারকে প্রধানত দুটি ভাগে ভাগ করা যায়:

১. 2-Tier Architecture

এই মডেলে ক্লায়েন্ট সরাসরি সার্ভারের সাথে যোগাযোগ করে। এখানে দুটি স্তর থাকে: Client Layer এবং Server Layer (যেটি প্রসেসিং এবং ডেটাবেস দুটোর কাজই করে)। ছোট অ্যাপ্লিকেশন এবং কম ইউজারের সফটওয়্যারের জন্য এটি উপযোগী।

Client Tier
Presentation / UI
Direct DB Query
Database Server
Processing & Data Layer

২. 3-Tier Architecture

পারফরম্যান্স, সিকিউরিটি এবং স্কেলাবিলিটি বা সিস্টেম বড় করার সুবিধার জন্য 3-Tier আর্কিটেকচার ব্যবহার করা হয়। এটি বর্তমানে ওয়েব অ্যাপ্লিকেশনে সবচেয়ে জনপ্রিয় মডেল। এখানে সিস্টেমকে তিনটি লেয়ারে ভাগ করা হয়:

  • Presentation Layer (Client): ইউজার ইন্টারফেস।
  • Application Layer (Business Logic): সার্ভার যা রিকোয়েস্ট প্রসেস করে এবং লজিক অ্যাপ্লাই করে।
  • Data Layer (Database Server): যেখানে মূল ডেটা সংরক্ষিত থাকে।

এখানে ক্লায়েন্ট সরাসরি ডেটাবেসের সাথে কানেক্ট হতে পারে না, ফলে ডেটাবেস অনেক বেশি নিরাপদ থাকে।

ব্যবহারিক ক্ষেত্র (Applications)

Web Browsing

ব্রাউজার (Client) ওয়েব সার্ভার থেকে ওয়েবপেজ রিকোয়েস্ট করে ডেলিভারি নেয়。

Email Systems

মেইল ক্লায়েন্ট (যেমন Outlook) মেইল সার্ভার থেকে মেসেজ রিসিভ ও সেন্ড করে。

Cloud Storage

ফাইল সিঙ্ক, আপলোড বা ডাউনলোডের জন্য ক্লাউড স্টোরেজ সার্ভার ও ক্লায়েন্ট অ্যাপ。

সুবিধা ও সীমাবদ্ধতা (Advantages & Limitations)

সুবিধাসমূহ (Advantages)

  • Centralized Data: ডেটা এবং সার্ভিস এক জায়গায় থাকে বলে ম্যানেজমেন্ট এবং আপডেট অনেক সহজ।
  • Security: সার্ভার এন্ডে শক্তিশালী সিকিউরিটি এবং অ্যাক্সেস কন্ট্রোল ব্যবহার করা যায়।
  • Scalability: একই সার্ভিস একসাথে অসংখ্য ক্লায়েন্টকে দেওয়া যায়।
  • Consistency: ক্লায়েন্ট একটিমাত্র বিশ্বস্ত সোর্স (Trusted Source) থেকে ডেটা পায়।

সীমাবদ্ধতা (Limitations)

  • Server Dependency: সার্ভার ফেল করলে (SPOF - Single Point of Failure) পুরো সার্ভিস বন্ধ হয়ে যায়।
  • Bottleneck: একসাথে অত্যাধিক রিকোয়েস্ট এলে সার্ভার স্লো হয়ে যেতে পারে।
  • High Cost: শক্তিশালী হার্ডওয়্যার এবং ব্যাকআপ সলিউশন মেইনটেইন করা বেশ ব্যয়বহুল।
  • Network Reliant: ইন্টারনেট বা নেটওয়ার্ক কানেকশন ছাড়াই এটি অচল।

উপসংহার: Client-Server Model-ই হলো আধুনিক ইন্টারনেট এবং এন্টারপ্রাইজ নেটওয়ার্কের মেরুদণ্ড। সিকিউরিটি, ডেটা ম্যানেজমেন্ট এবং স্কেলাবিলিটি—সবকিছু মিলিয়ে বড় সিস্টেম পরিচালনার জন্য এর বিকল্প নেই।
