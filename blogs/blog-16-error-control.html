Error Control in Data Link Layer

February 05, 2026 | Networking

Error Control What is?

Data Link Layer-এ Error Control নিশ্চিত করে যে Sender কর্তৃক পাঠানো Frame-গুলো Receiver-এর কাছে সঠিকভাবে ও Reliable ভাবে পৌঁছায়। এটি মূলত তিনটি Task করে:

  • Data Transmission-এর সময় ঘটা Error শনাক্ত করা।
  • পুনরায় Transmission-এর মাধ্যমে Error সংশোধন করা।
  • Noise-যুক্ত Channel-এ Reliable Frame Delivery নিশ্চিত করা।

Error Control-এর প্রাথমিক Method হলো Automatic Repeat Request (ARQ)। ARQ-এ যদি Error শনাক্ত হয় বা ACK পাওয়া না যায়, তাহলে Sender Frame-টি পুনরায় পাঠায়।

Error Control-এর দুটি Method

Error Detection

Transmitted Data-য় Error শনাক্ত করার প্রক্রিয়া। যোগাযোগ ব্যবস্থায় Transmitter থেকে Receiver-এ Data পাঠানোর সময় Noise বা অন্যান্য সমস্যার Reason ে Error হতে পারে। এটি নষ্ট, বিকৃত বা অস্পষ্ট Data আরও Process করার আগে Symbol িত করার Technique ব্যবহার করে।

Error Correction

শনাক্ত হওয়া Error সংশোধন করে মূল ত্রু Team ুক্ত Data পুনরুদ্ধার করার প্রক্রিয়া। এটি অতিরিক্ত Information ব্যবহার করে বা পুনরায় Transmission-এর মাধ্যমে সঠিক Data পুনর্ Structure করে। তবে Error Correction Technique সাধারণত জটিল, ব্যয়বহুল ও Resource-intensive হওয়ায় এটি বেছে বেছে ব্যবহার করা হয়।

Error Control-এর Technique সমূহ (Techniques)

1. Stop-and-Wait ARQ

এই Method তে Sender একটি Data Frame পাঠায় এবং Receiver-এর কাছ থেকে ACK (Acknowledgment) পাওয়ার আগ পর্যন্ত অপেক্ষা করে।

  • একটি Frame পাঠিয়ে ACK না পাওয়া পর্যন্ত অপেক্ষা করে।
  • Timeout-এর মধ্যে ACK না পেলে Frame পুনরায় পাঠানো হয়।
  • সকল Frame সফলভাবে পৌঁছানো পর্যন্ত এই প্রক্রিয়া চলে।

2. Sliding Window ARQ

এটি Continuous Data Transmission-এর জন্য ব্যবহৃত হয়। Sender প্রতিটি Frame-এর ACK-এর জন্য অপেক্ষা না করেই Multiple Frame পাঠাতে পারে, যা Channel ব্যবহারের দক্ষতা বাড়ায়।

Sliding Window ARQ প্রধানত দুই Type ের:

Go-Back-N ARQ

  • Sender একটি নির্দিষ্ট Window Size পর্যন্ত ACK ছাড়াই Frame পাঠাতে পারে।
  • Receiver সঠিকভাবে Frame পেলে ACK পাঠায়।
  • কোনো Frame হারিয়ে গেলে বা Error যুক্ত হলে Receiver সেই Frame এবং Next সব Frame বাদ দেয়।
  • Sender তখন সেই Error পূর্ণ Frame এবং তার পরে পাঠানো সব Frame পুনরায় পাঠায়।
  • Error না থাকলে এটি সাধারণ Sliding Window Mechanism হিসেবে Task করে।
  • বাস্তবায়ন সহজ, তবে অপ্রয়োজনীয় পুনরায় পাঠানো হতে পারে।

Selective Repeat ARQ

  • Go-Back-N-এর মতো Multiple Frame পুনরায় না পাঠিয়ে এটি দক্ষতা Advanced করে।
  • Sender শুধুমাত্র সেই Frame-গুলো পুনরায় পাঠায় যেগুলোর জন্য NAK পাওয়া গেছে বা Timeout হয়েছে।
  • প্রতিটি Frame Separate ভাবে স্বীকৃত হয়, যা সঠিকভাবে প্রাপ্ত Frame-গুলো গ্রহণ ও সংরক্ষণ করার Advantage দেয়।
  • অতিরিক্ত Buffering ও Tracking-এর প্রয়োজনে বাস্তবায়ন জটিল, যা সরল ARQ-এর তৃলনায় এর ব্যবহার সীমিত করে।
মূল পার্থক্য: Go-Back-N ARQ-এ Error পূর্ণ Frame সহ Next সব Frame পুনরায় পাঠানো হয়, কিন্তু Selective Repeat ARQ-এ শুধুমাত্র ক্ষতিগ্রস্ত Frame টিই পাঠানো হয়, যা এটিকে আরও দক্ষ করে তোলে।

Summary

Error Control Data Link Layer-এর একটি অপরিহার্য কার্যকারিতা। Stop-and-Wait ARQ সহজ কিন্তু Slow, আর Sliding Window ARQ (Go-Back-N ও Selective Repeat) High দক্ষ ও Modern Network-এ ব্যাপকভাবে ব্যবহৃত হয়।
