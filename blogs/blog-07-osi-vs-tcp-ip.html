Difference between OSI and TCP/IP Models

Jan 02, 2026 | Protocol

ভূমিকা

Computer Networking জগতে OSI (Open Systems Interconnection) এবং TCP/IP (Transmission Control Protocol / Internet Protocol)— এই দুটি Model-ই Information আদান-প্রদানের মূল ভিত্তি। যদিও এরা Separate Purposeে Creation হয়েছে, তবে এদের প্রধান Task হলো একটি Network-এর মাধ্যMay দুটি Device-এর মধ্যে Data কীভাবে প্রবাহিত হবে তা সংজ্ঞায়িত করা। OSI একটি Theoretical Reference Model হিসেবে Task করে, অন্যDirectionে TCP/IP হলো Modern Internet-এর বাস্তবসম্মত ভিত্তি।

OSI Model (৭টি Layer)

OSI Model-টি ১৯৭০-এর দশকের শেষDirectionে ISO দ্বারা Creation একটি Theoretical Framework। এতে মোট ৭টি Layer থাকে:

1. Physical Layer

  • Physical Media-র (তার, Connector) মাধ্যMay Raw Bits ট্রান্সমিট করে।
  • Digital Data-কে Electrical, Optical বা Radio Signal-এ রূপান্তর করে।
  • Data Rate, Modulation এবং Physical Topology (যেমন: Bus, Star) সংজ্ঞায়িত করে।
  • Hub এবং Repeater এই Layer-এ Task করে।

2. Data Link Layer

  • Node-to-Node Reliable Data Transfer এবং Framing নিশ্চিত করে।
  • Device শনাক্তকরণের জন্য MAC Address ব্যবহার করে।
  • Error Detection (FCS) এবং Flow Control Manage করে।
  • Switch এবং Bridge এই Layer-এ Task করে।

3. Network Layer

  • Packet-এর Logical Addressing এবং IP Routing Handle করে।
  • IP Address ব্যবহার করে Destinationে পৌঁছানোর সেরা Path Determination করে।
  • Packet Forwarding এবং Routing Table Manage করে।
  • Router এই Layer-এ Task করে।

4. Transport Layer

  • End-to-End Reliable বা Fast Data Delivery নিশ্চিত করে।
  • TCP (Reliable/Connection-oriented) এবং UDP (Fast/Connectionless) Protocol ব্যবহার করে।
  • Data Segmentation, Sequencing এবং Re-assembly করে।
  • Flow Control এবং Error Recovery প্রদান করে।

5. Session Layer

  • Device-এর মধ্যে Session Creation, Operation এবং সমাপ্ত করে।
  • এটি Dialogue Control (Half-duplex / Full-duplex) করে।
  • Session Checkpoint এবং Authentication বজায় রাখে।
  • RPC এবং NetBIOS-এর মতো Protocol এখানে Task করে।

6. Presentation Layer

  • Data Translation, Encryption এবং Formatting Handle করে।
  • Data Format Convert (Encoding/Decoding) করে যাতে Application Layer তা বুঝতে পারে।
  • Securityর জন্য Encryption/Decryption (SSL/TLS) প্রয়োগ করে।
  • দক্ষ Transmission-এর জন্য Data Compression করে।

7. Application Layer

  • সরাসরি End-user Application-কে Network Service প্রদান করে।
  • HTTP, FTP, SMTP, DNS এবং SNMP-এর মতো Protocol সমর্থন করে।
  • Email, Web Browsing এবং File Transfer-এর মতো অনলাইন সেবাগুলো নিশ্চিত করে।

TCP/IP Model (৫টি Layer)

TCP/IP Model-টি Internet-এর জন্য Creation একটি Practical Framework। এটি বর্তমানে বিশ্বজুড়ে ব্যব্যহৃত হয় এবং এতে নিচের ৫টি Layer-এর Detailed Task রয়েছে:

1. Physical Layer

  • Physical Media-র মাধ্যMay Raw Bits ট্রান্সমিট করে।
  • Physical Hardware যেমন: Cable, Signal এবং Connector নিয়ে Task করে।
  • Hub, Repeater এবং NIC এই Layer-এর অংশ।

2. Data Link Layer

  • Node-to-Node যোগাযোগ এবং Framing নিশ্চিত করে।
  • Device শনাক্তকরণের জন্য MAC Address ব্যবহার করে।
  • Framing, Error Detection (FCS) এবং Flow Control-এর জন্য দায়ী।
  • Physical Medium-এ (Ethernet, Wi-Fi) Access Handle করে।
  • Switch এই Layer-এ Task করে।

3. Network (Internet) Layer

  • Logical Addressing এবং Packet Routing Handle করে।
  • IP Addressing (IPv4/IPv6) ব্যবহার করে।
  • Destinationে পৌঁছানোর জন্য সেরা Path Determination করে।
  • Packet Routing এবং Fragmentation সম্পন্ন করে।
  • প্রধান Protocol: IP, ICMP, ARP, RARP।

4. Transport Layer

  • End-to-End যোগাযোগ এবং Reliableতা নিশ্চিত করে।
  • TCP (Reliable, Connection-oriented) এবং UDP (Fast, Connectionless) ব্যবহার করে।
  • Data Segmentation, Sequencing and Re-assembly করে।
  • Error Detection, Correction এবং Flow Control প্রদান করে।

5. Application Layer

  • End-user Application-কে সরাসরি Network Service প্রদান করে।
  • HTTP, FTP, SMTP, DNS, DHCP এবং SNMP Protocol সমর্থন করে।
  • Data Formatting এবং Communication Service প্রদানের জন্য দায়ী।
  • এটি OSI Model-এর Session, Presentation এবং Application Layer-এর Taskগুলো একসাথে সম্পন্ন করে।

OSI এবং TCP/IP Model-এর মধ্যে Detailed পার্থক্য

Feature OSI Model TCP/IP Model
Layer সংখ্যা ৭টি Layer থাকে। ৫টি (বা ৪টি) Layer থাকে।
Session ও Presentation Layer Separate Session এবং Presentation Layer থাকে। Separate কোনো Layer নেই; এগুলো Application Layer-এর অংশ।
প্রকৃতি এটি একটি Theoretical Reference Model। এটি একটি Practical Implementation Model।
Creation ও বিকাশ ISO দ্বারা Internet-এর ব্যাপক প্রসারের আগে Creation। DoD (Department of Defense) দ্বারা Internet-এর জন্য Creation।
নির্ভরশীলতা প্রতিটি Layer কঠোরভাবে সীমানা নির্দিষ্ট এবং স্বাধীন। Layer-গুলো অনেক High Integrated এবং সীমানা কম কঠোর।
Protocol কোনো নির্দিষ্ট Protocol Determination করে না, শুধু Function বর্ণনা করে। নির্দিষ্ট Protocol (TCP, IP, UDP, HTTP) সংজ্ঞায়িত করে।
Creationর ক্রম আগে Model Creation করা হয়েছে, পরে Protocol। আগে Protocol Creation করা হয়েছে, পরে Model।
ব্যবহার মূলত শিক্ষাদান এবং Network Design-এর Reference হিসেবে ব্যবহৃত হয়। Internet-এ যোগাযোগের জন্য প্রধান Protocol Stack হিসেবে ব্যবহৃত হয়।

Summary

सहज কথায়, OSI Model আমাদের শেখায় কীভাবে Protocol Task করা উচিত, অন্যDirectionে TCP/IP Model Internetের বাস্তব যোগাযোগ ব্যবস্থা নিশ্চিত করে। কোনো Network Engineer যদি সিস্টেMay কোনো সমস্যা খুঁজে বের করতে চান (Troubleshooting), তবে তিনি সাধারণত OSI Model Layerকেই রেফারেন্স হিসেবে ব্যবহার করেন।
