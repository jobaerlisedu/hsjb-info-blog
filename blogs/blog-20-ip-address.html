What is an IP Address?

February 27, 2026 | Networking

IP Address (Internet Protocol Address) হলো একটি Unique গাণিতিক Label যা কোনো Computer Network-এ যুক্ত থাকা প্রতিটি Device-এর Identity হিসেবে ব্যবহৃত হয়। সহজ কথায়, এটি আপনার Device-এর একটি Digital বাড়ির Address, যার মাধ্যমে Internet-এ Data আদান-প্রদান করা সম্ভব হয়।

IP Address মূলত দুটি প্রধান Task করে:
  • 1. Network-এ Device-টিকে শনাক্ত করা।
  • 2. Device-এর অবস্থান খুঁজে বের করে অন্য Device-এর সাথে যোগাযোগ স্থাপন করা।

IP Address-এর Component সমূহ (Components)

একটি IP Address প্রধানত তিনটি অংশে বিভক্ত থাকে:

  • Network Portion: এটি Device-টি কোন Network-এর অন্তর্ভুক্ত তা শনাক্ত করে।
  • Host Portion: এটি ওই Network-এর নির্দিষ্ট Device-টিকে শনাক্ত করে।
  • Subnet Mask: এটি নির্ধারণ করে IP Address-এর কতটুকু অংশ Network-এর এবং কতটুকু Host-এর।

IP Address-এর Classification

1. Public vs Private IP

Public IP Address

Internet-এ সরাসরি Access করার জন্য এটি ব্যবহৃত হয়। বিশ্বজুড়ে এটি Unique হয় এবং ISP (Internet Service Provider) এটি প্রদান করে।

Private IP Address

এটি Local Network-এর (যেমন- আপনার বাড়ি বা Office) ভেতরে যোগাযোগের জন্য ব্যবহৃত হয়। এটি Internet-এ সরাসরি Routable নয়।

2. IPv4 vs IPv6

IPv4

এটি 32-Bit Addressing System, যা ডট দিয়ে Separate করা 4টি সংখ্যার সেট (যেমন: 192.168.1.1)। এতে প্রায় 4 বিলিয়ন Address সম্ভব।

IPv6

IP সংকটের কারণ ে এটি তৈরি করা হয়েছে। এটি 128-Bit Addressing System এবং Hexadecimal সংখ্যায় প্রকাশ করা হয়।

3. Static vs Dynamic IP

  • Static IP: এটি স্থায়ীভাবে কোনো Device-এ বরাদ্দ করা হয় (যেমন Server-এর জন্য)।
  • Dynamic IP: এটি সাময়িকভাবে DHCP Server দ্বারা বরাদ্দ করা হয় এবং Rule িত পরিবর্তিত হতে পারে।

4. কার্যের ভিত্তিতে IP Address

Unicast Address

একজন Sender থেকে একজন নির্দিষ্ট Receiver কাছে Data পাঠানো। এটি সবচেয়ে সাধারণ যোগাযোগ Method।

Broadcast Address

একটি Device-এর মাধ্যমে Network-এর সকল Device-এ Message পাঠানো। IPv6-এ এটি ব্যবহৃত হয় না।

Multicast Address

একই সাথে Multiple নির্দিষ্ট Device-এর Group-এ Data পাঠানো (যেমন: লাইভ স্ Tree মিং)।

Anycast Address

একই IP Share করা Group থেকে ভৌগোলিকভাবে সবচেয়ে কাছের Device-এ Information পাঠানো।

IPv4 Address-এর Class-সমূহ (Classes)

সহজে ব্যবস্থাপনার জন্য IPv4 Address-কে 5টি Class-এ ভাগ করা হয়েছে:

Class IP Range ব্যবহারের Field
Class A 1.0.0.0 — 127.255.255.255 বিশাল Network (যেমন: Multinational Company)।
Class B 128.0.0.0 — 191.255.255.255 Medium আকারের Network।
Class C 192.0.0.0 — 223.255.255.255 ছোট ব্যবসা বা Home Network।
Class D 224.0.0.0 — 239.255.255.255 মাল্টিকাস্ট গ্রুপের জন্য সংরক্ষিত।
Class E 240.0.0.0 — 255.255.255.255 Research ও ভবিষ্যতের ব্যবহারের জন্য।

বিশেষ IP Address (Special IP Addresses)

  • Loopback Address / Localhost: 127.0.0.1 (127.0.0.1) Address-টি নিজের Device-এর Connectivity Test করতে ব্যবহৃত হয়।
  • Broadcast Address: একটি Network-এর সকল Device-এ Information পাঠাতে ব্যবহৃত হয় (যেমন: 192.168.1.255)।
  • 0.0.0.0: এটি একটি Non-routable Address যা Default Route হিসেবে ব্যবহৃত হয়।

IP Address কীভাবে কাজ করে?

Unique Identification

প্রতিটি Device-এ একটি IP থাকায় Router Information পাঠানোর সময় সঠিক Destination চিনতে পারে।

Data Packeting

যখন কোনো Data পাঠানো হয়, তখন সেটি ছোট ছোট Packet-এ বিভক্ত হয়। প্রতিটি Packet-এ Sender এবং Receiver-এর IP থাকে।

Routing

Router-গুলো IP Address দেখে নির্ধারণ করে কোন Path ে Data পাঠালে Fast পৌঁছাবে।

NAT (Network Address Translation)

আপনার Local Device-এর Private IP-কে Router Public IP-তে রূপান্তর করে Internet-এ পাঠায়।

বাস্তব Example: Email পাঠানো

নিউইয়র্ক থেকে টোকিওতে একজন বন্ধুকে Email পাঠানোর Step-গুলো:

  1. আপনার Email Client Data-কে Packet-এ ভাগ করে এবং তাতে আপনার ও আপনার বন্ধুর Mail Server-এর IP যুক্ত করে।
  2. আপনার Router Packet-গুলোকে ISP-র কাছে পাঠায়।
  3. Router-গুলো Global Network-এর মাধ্যমে Packet-গুলোকে জাপানে টোকিও Server-এর কাছে পৌঁছে দেয়।
  4. টোকিও Mail Server Packet-গুলোকে একত্র করে এবং আপনার বন্ধুর Device-এ Email-টি পৌঁছে দেয়।

IP Address Check করার উপায়

  • Windows: Command Prompt অপেন করে `ipconfig` লিখে Enter দিন।
  • Mac/iPhone: Network Settings বা Wi-Fi সেটিংস থেকে IP Check করা যায়।
  • Public IP: গুগল সার্চে "What is my IP" লিখলে আপনার Public IP দেখা যায়।

Security এবং সুরক্ষা

IP Address ওপেন থাকলে বিভিন্ন Risk যেমন- IP Spoofing, DDoS Attacks বা Man-in-the-Middle Attack হতে পারে।

সুরক্ষার উপায়:

  • VPN ব্যবহার করা: এটি আপনার আসল IP লুকিয়ে ফেলে।
  • Proxy Server: এটি আপনার ও Internet-এর মাঝে মধ্যস্থতাকারী হিসেবে Task করে।
  • Firewall: অননুমোদিত Connection Block করার জন্য Firewall ব্যবহার করুন।
