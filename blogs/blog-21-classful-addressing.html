Classful IP Addressing

February ২৭, 2026 | Networking

Classful IP Addressing ছিল IP Address Assign করার একটি প্রাথমিক Method (১৯৮১-১৯৯৩), যেখানে IPv4 Address Space-কে পাঁচটি নির্দিষ্ট Class-এ বিভক্ত করা হতো। প্রতিটি IP Address-এর অগ্রণী Bit-গুলোর ওপর ভিত্তি করে তার Class Fixed হতো, যা Network এবং Host অংশের বিভাজন Determination করত।

বর্তমানে Classফুল Addressিংয়ের পরিবর্তে CIDR (Classless Inter-Domain Routing) ব্যবহৃত হয়, যা আরও Flexible এবং দক্ষ।

IP Address-এর Notations-সমূহ

Dotted Decimal

এটি মানুষের জন্য পাঠযোগ্য Format, যেখানে ৩২-Bit Binary Address-কে চারটি ৮-Bit Segment-এ (Octet) ভাগ করে ডট দিয়ে লেখা হয়।

  • প্রতিটি সেগMayন্ট ০ থেকে ২৫৫ পর্যন্ত হয়।
  • Example: 192.168.1.1

Hexadecimal

Networking এবং Debugging-এর জন্য ব্যবহৃত হয়, যেখানে IP Address-কে Base-16 সংখ্যায় লেখা হয়।

  • প্রতিটি বাইটকে দুটি হেক্সাডেসিমাল ডিজিটে প্রকাশ করা হয়।
  • বাইনারির তুলনায় এটি সংক্ষিপ্ত।

কেন Classful Addressing প্রয়োজন ছিল?

  • সহজ বরাদ্দ: নির্দিষ্ট Class-এর মাধ্যMay IP বরাদ্দ করা সহজ ছিল।
  • Fast Routing: Router-গুলো প্রথম Bit দেখেই Class এবং Network ID বুঝতে পারত।
  • Scalability: বড় এবং ছোট উভয় Typeের Network-এর জন্য Separate Class (A, B, C) ছিল।
  • Standardization: সমস্ত Device-এর মধ্যে সামঞ্জস্য নিশ্চিত করত।

IP Address Management

IP Address-গুলো বিশ্বব্যাপী Internet Assigned Numbers Authority (IANA) এবং Regional Internet Registries (RIRs) দ্বারা পরিচালিত হয়। তারা নিশ্চিত করে যে IP বরাদ্দ যেন সুশৃঙ্খল এবং Unique হয়।

Classful Addressing-এর Structure (Structure)

নিচের Tableে প্রতিটি Classের Bit বিভাজন এবং Address রেঞ্জ দেখানো হলো:

Class Leading Bits Net ID (Bit) Host ID (Bit) মোট Network ব্যবহারযোগ্য Host
Class A 0 8 24 128 (2⁷) 16,777,214
Class B 10 16 16 16,384 (2¹⁴) 65,534
Class C 110 24 8 2,097,152 (2²¹) 254
Class D 1110 - - - -
Class E 1111 - - - -

দ্রষ্টব্য: Class A-তে মূলত ১২৭টি Network ধরা হয় (0.0.0.0 এবং 127.0.0.0 বিশেষ Reasonে বাদ দেওয়া হয়)।

IP Address-এর Classসমূহ (Detailed)

৩২-Bit IP Address-কে নিম্নোক্ত ৫টি Class-এ ভাগ করা হয়:

Class A

অনেক বিশাল সংখ্যক Host প্রয়োজন এমন Network-এর জন্য ব্যবহৃত হয়।

  • প্রথম Bit: সব সময় '0' থাকে।
  • Address রেঞ্জ: 0.0.0.0 থেকে 127.255.255.255
  • Network আইডি: ৮ Bit | হোস্ট আইডি: ২৪ Bit
  • ব্যবহারযোগ্য হোস্ট: ১,৬৭,৭৭,২১৪ টি (প্রতি Networkে)।
  • সাবনেট মাস্ক: 255.0.0.0
Class B

Medium থেকে বড় Sizeের Network-এর জন্য বরাদ্দ করা হয়।

  • প্রথম দুই Bit: সব সময় '10' থাকে।
  • Address রেঞ্জ: 128.0.0.0 থেকে 191.255.255.255
  • Network আইডি: ১৬ Bit | হোস্ট আইডি: ১৬ Bit
  • ব্যবহারযোগ্য হোস্ট: ৬৫,৫৩৪ টি।
  • সাবনেট মাস্ক: 255.255.0.0
Class C

ছোট Sizeের Network-এর জন্য ব্যবহৃত হয়।

  • প্রথম তিন Bit: সব সময় '110' থাকে।
  • Address রেঞ্জ: 192.0.0.0 থেকে 223.255.255.255
  • Network আইডি: ২৪ Bit | হোস্ট আইডি: ৮ Bit
  • ব্যবহারযোগ্য হোস্ট: ২৫৪ টি।
  • সাবনেট মাস্ক: 255.255.255.0
Class D

মাল্টিকাস্ট (Multicast) যোগাযোগের জন্য সংরক্ষিত।

  • প্রথম চার Bit: সব সময় '1110' থাকে।
  • Address রেঞ্জ: 224.0.0.0 থেকে 239.255.255.255
  • কোনো Network বা হোস্ট আইডি বিভাজন নেই।
Class E

Research এবং Testমূলক (Experimental) Taskের জন্য সংরক্ষিত।

  • প্রথম চার Bit: সব সময় '1111' থাকে।
  • Address রেঞ্জ: 240.0.0.0 থেকে 255.255.255.255

বিশেষ IP Address Range

  • 127.0.0.0 - 127.255.255.255: Loopback Address, নিজের Computer Test করার জন্য ব্যবহৃত হয়।
  • 169.254.0.0 - 169.254.255.255: Link-Local Address, যখন DHCP Server থেকে IP পাওয়া যায় না।
  • 0.0.0.0: এটি বর্তমান Network-কে বোঝায়।

ID Assign করার Ruleাবলি

Host ID

  • প্রতিটি Host ID অবশ্যই Unique হতে হবে।
  • সব Bit '0' হতে পারবে না (এটি Network Address)।
  • সব Bit '1' হতে পারবে না (এটি Broadcast Address)।

Network ID

  • Network ID 127 দিয়ে শুরু হতে পারবে না।
  • সব Bit '1' Broadcast-এর জন্য সংরক্ষিত।
  • সব Bit '0' Local Network-এর জন্য।

Classful Addressing-এর সমস্যাসমূহ

Classful Addressing-এর প্রধান সমস্যা ছিল IP Address-এর অপচয়। Exampleস্বরূপ:

  • Class A এবং B Network-এ অনেক High IP অপচয় হতো।
  • Class C Network-এ Host সংখ্যা খুব কম (২৫৪ টি) ছিল যা বড় প্রতিষ্ঠানের জন্য Sufficient নয়।
  • Addressing-এ Flexibleতা এবং Scalability-র অভাব ছিল।

এই সমস্যাগুলোর Reasonেই ১৯৯৩ সালে CIDR প্রবর্তন করা হয়।

Classful vs Classless Addressing

Feature Classful Addressing Classless Addressing (CIDR)
বিভাজন স্থায়ী Class (A, B, C, D, E) ভিত্তিক। Flexible Prefix Length ভিত্তিক।
Subnetting Default-ভাবে Subnetting Support করে না। Subnetting এবং VLSM Support করে।
IP অপচয় অত্যধিক IP অপচয় ঘটে। IP Address-এর দক্ষ ব্যবহার নিশ্চিত করে।
Routing Table বড় Routing Table Creation হয়। Route Aggregation-এর মাধ্যMay ছোট Table Creation হয়।
Scalability খুবই সীমিত। Modern Network-এর জন্য অত্যন্ত Scalable।
