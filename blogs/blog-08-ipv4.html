What is IPv4?

Networking January 02, 2026 Jobaer Hossain

Introduction

IPv4 stands for Internet Protocol version 4. It is the fourth version of the Internet Protocol (IP) and the first version to be widely deployed. IPv4 is a connectionless protocol used in packet-switched layer networks, such as Ethernet. It provides logical addressing for devices on a network.

Features of IPv4

  • Connectionless Protocol
  • Simple and flexible
  • Allows creation of virtual networks
  • Low memory requirement
  • Supported by millions of devices
  • Provides video libraries and conferences

IPv4 Address Format

IPv4 uses 32-bit addresses, written in dotted decimal notation (four octets separated by dots), e.g., 192.168.1.1.

Each octet ranges from 0 to 255, providing approximately 4.3 billion unique addresses.

IPv4 Header Format

The IPv4 packet header consists of 20 bytes (minimum) with the following fields:

  • Version (4 bits): Indicates IP version (4 for IPv4)
  • Header Length (IHL, 4 bits): Length of header in 32-bit words
  • Type of Service (8 bits): Priority and QoS
  • Total Length (16 bits): Total packet size
  • Identification (16 bits): For fragmentation
  • Flags (3 bits): Fragmentation control
  • Fragment Offset (13 bits): Position in original packet
  • Time to Live (TTL, 8 bits): Prevents infinite loops
  • Protocol (8 bits): Upper-layer protocol (TCP, UDP, etc.)
  • Header Checksum (16 bits): Error detection
  • Source IP Address (32 bits): Sender address
  • Destination IP Address (32 bits): Receiver address
  • Options (variable): Optional fields

Classes of IPv4 Addresses

IPv4 addresses are divided into five classes based on the first octet:

Class First Octet Range Purpose
A 0 – 127 Large networks (unicast)
B 128 – 191 Medium networks (unicast)
C 192 – 223 Small networks (unicast)
D 224 – 239 Multicast
E 240 – 255 Reserved / Experimental

Limitations of IPv4

  • Address exhaustion (only ~4.3 billion addresses)
  • No built-in security (IPsec added later)
  • Complex configuration
  • Lack of auto-configuration
  • No quality of service (QoS) support in header

Why IPv6 Was Introduced

Due to the rapid growth of the Internet, IPv4 addresses were running out. IPv6 was developed to provide:

  • 128-bit addresses (nearly unlimited)
  • Built-in security (IPsec mandatory)
  • Auto-configuration
  • Better support for mobile devices
  • Improved routing and QoS

Summary

IPv4 is still widely used today despite its limitations. It forms the backbone of the current Internet, but the transition to IPv6 is ongoing to accommodate future growth.
