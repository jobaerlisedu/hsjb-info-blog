Piggybacking: Computer Network

February ১০, 2026 | Networking

Piggybacking What is?

Piggybacking হলো একটি Technique যেখানে Receiver তার ACK (Acknowledgment) পাঠানো বিলম্বিত করে এবং Next বাইরে যাওয়া Data Packetের সাথে একত্রে সংযুক্ত করে পাঠায়। এর ফলে Separate Control ফ্রেMayর সংখ্যা কMay এবং Network দক্ষতা বাড়ে।

  • Sliding Window Protocol Sender ACK পাওয়ার আগেই Multiple Packet পাঠাতে দেয়, যা Throughput বাড়ায়।
  • Sender ও Receiver উভয়ই আগত ও বাহির হওয়া Packet ধরে রাখতে সীমিত Buffer রাখে।
  • নির্দিষ্ট সময়ের মধ্যে ACK না পেলে অস্বীকৃত Packet-গুলো পুনরায় পাঠানো হয়।

Network Efficiency কীভাবে বাড়াবেন?

Network Efficiency Full-Duplex Transmission-এর মাধ্যMay বাড়ানো যায়, যা একই সময়ে দুই Directionে যোগাযোগ সম্ভব করে। এটি Simplex বা Half-Duplex-এর মতো নয়, এখানে উভয় প্রান্ত একই সময়ে Data পাঠাতে ও গ্রহণ করতে পারে।

Full-Duplex Transmission দুটি উপায়ে অর্জন করা সম্ভব:

দুটি Separate Channel

একটি Channel Data পাঠানোর জন্য এবং অপরটি গ্রহণের জন্য ব্যবহৃত হয়। তবে যখন একDirectionের Traffic কম থাকে, তখন Bandwidth অপচয় হয়।

Piggybacking

একটি আরও দক্ষ Method যেখানে Data ও ACK একই Channel Share করে। Receiver ACK বিলম্বিত করে Next Data Frame-এ সংযুক্ত করে পাঠায়।

Piggybacking কীভাবে Task করে?

Piggybacking-এ Separate দুটি Message (DATA + ACK) না পাঠিয়ে একটি মাত্র Combined Message পাঠানো হয়:

Host A
Data আছে + ACK আছে

DATA + ACK ➜
◀ DATA + ACK

Host B
Data আছে + ACK আছে

  • যদি Host-এর কাছে Data ও ACK উভয়ই থাকে: একটি Frame-এ উভয়ই পাঠায়।
  • যদি শুধু ACK থাকে: Host কিছুক্ষণ অপেক্ষা করে Data প্রস্তুত কিনা দেখে, না হলে ACK Separateভাবে পাঠায়।
  • যদি শুধু Data থাকে: সর্বশেষ ACKটি Data Frame-এ অন্তর্ভুক্ত করে পাঠায়।

Piggybacking উপলব্ধ Channel Bandwidth-এর আরও ভালো ব্যবহার নিশ্চিত করে Data ও ACK একটি মাত্র Frame-এ মিলিয়ে দেওয়ার মাধ্যMay — Separate ACK Frame পাঠানোর প্রয়োজনীয়তা দূর করে। এটি Control Overhead কমায়, Transmission Cost হ্রাস করে এবং বিশেষত দ্বিমুখী যোগাযোগে Data Transfer-এর Latency Advanced করে।

Piggybacking-এর সীমাবদ্ধতা

সতর্কতা: Piggybacking Protocol-এ অতিরিক্ত Complexity আনে। Sender-কে সিদ্ধান্ত নিতে হয় ACK পাঠানোর আগে কতক্ষণ অপেক্ষা করবে। যদি Data Link Layer ACK High সময় ধরে অপেক্ষা করে, তাহলে Sender Frame-টি হারিয়ে গেছে মনে করে পুনরায় পাঠাতে পারে — যা দক্ষতা কমিয়ে দেয়।

Summary

Piggybacking হলো একটি চতুর Technique যা Network-এ Separate ACK Frame-এর সংখ্যা কমিয়ে দক্ষতা Advanced করে। বিশেষত Full-Duplex যোগাযোগে এটি Channel Bandwidth-এর সর্বোত্তম ব্যবহার নিশ্চিত করে। তবে ACK বিলম্বের সঠিক সামঞ্জস্য না হলে অপ্রয়োজনীয় পুনরায় পাঠানো হতে পারে।
