User Datagram Protocol (UDP)

March 09, 2026 Networking

User Datagram Protocol (UDP) হলো ইন্টারনেট প্রোটোকল (IP)-এর একটি Transport Layer প্রোটোকল যা প্রসেসগুলোর মধ্যে দ্রুত, কানেকশনলেস (connectionless) এবং লাইটওয়েট যোগাযোগ প্রদান করে। এটি ডেটা ডেলিভারি, অর্ডার বা এরর চেকিংয়ের কোনো গ্যারান্টি দেয় না, যার কারণে এটি রিয়েল-টাইম এবং সময়-সংবেদনশীল (time-sensitive) অ্যাপ্লিকেশন যেমন- ভিডিও স্ট্রিমিং, DNS এবং VoIP-এর জন্য অত্যন্ত উপযোগী।

UDP Header (ইউডিপি হেডার):

UDP হেডারটি মাত্র ৮ বাইট (8 bytes) দীর্ঘ এবং এর ঠিক পরেই ডেটা পে-লোড (data payload) থাকে। এটি ট্রান্সমিশনের জন্য প্রয়োজনীয় সমস্ত মৌলিক তথ্য ধারণ করে।

  • Notes: TCP-এর মতো নয়, UDP-তে Checksum ঐচ্ছিক (optional)। এটি কোনো এরর বা ফ্লো কন্ট্রোল প্রদান করে না বরং এরর রিপোর্টিংয়ের জন্য IP/ICMP-এর ওপর নির্ভর করে।

Applications of UDP (UDP এর ব্যবহার)

যেখানে লো ল্যাটেন্সি (low latency) অত্যন্ত গুরুত্বপূর্ণ এবং মাঝে মাঝে প্যাকেট লস হওয়া গ্রহণযোগ্য, সেখানে UDP বেশি ব্যবহৃত হয়:

  • DNS (Domain Name System): ডোমেইন নেম কোয়েরি ছোট হওয়ায় এবং দ্রুত রিপ্লাইয়ের প্রয়োজন থাকায় DNS দ্রুত কোয়েরি/রেসপন্সের জন্য UDP ব্যবহার করে।
  • DHCP (Dynamic Host Configuration Protocol): ডিভাইসে ডাইনামিকভাবে আইপি অ্যাড্রেস অ্যাসাইন করার জন্য ছোট কন্ট্রোল মেসেজ আদান-প্রদান করতে DHCP, UDP ব্যবহার করে।
  • VoIP (Voice over IP): রিয়েল-টাইম ভয়েস কমিউনিকেশনে UDP ব্যবহৃত হয়, কারণ এটি কিছুটা প্যাকেট লস সহ্য করতে পারে কিন্তু কোনো লেটেন্সি বা ডিলে (delay) না।
  • RIP (Routing Information Protocol): রাউটারগুলোর মধ্যে রুট আপডেটগুলো দক্ষতার সাথে পাঠাতে RIP, UDP ব্যবহার করে।
  • NTP (Network Time Protocol): ন্যূনতম ওভারহেডের সাথে নেটওয়ার্কের সিস্টেম ক্লকগুলো সিঙ্ক্রোনাইজ করতে NTP, UDP ব্যবহার করে।

UDP Pseudo Header (স্যুডো হেডার)

Checksum-এর নির্ভুলতা বাড়াতে UDP একটি স্যুডো হেডার (pseudo header) ব্যবহার করে যা Checksum ক্যালকুলেশনের সময় যুক্ত হয়, কিন্তু ট্রান্সমিট করা হয় না।

  • এটি প্যাকেটটি সঠিক হোস্ট এবং প্রোটোকল পোর্টে ডেলিভার হওয়া নিশ্চিত করে।

Use of UDP in DDoS Attacks

UDP Flood Attack হলো এক ধরনের ডিস্ট্রিবিউটেড ডিনায়েল অফ সার্ভিস (DDoS) অ্যাটাক যেখানে অ্যাটাকার ইউডিপি-এর কানেকশনলেস প্রকৃতির সুযোগ নিয়ে টার্গেট পোর্টে বিপুল সংখ্যক UDP প্যাকেট পাঠায়।

  • Attack Process: অ্যাটাকার ফেক (spoofed) আইপি অ্যাড্রেস ব্যবহার করে টার্গেটের এলোমেলো পোর্টগুলোতে প্রচুর UDP প্যাকেট পাঠায়। টার্গেট সিস্টেম সেই পোর্টগুলোতে কোনো অ্যাক্টিভ অ্যাপ্লিকেশন আছে কিনা তা চেক করে এবং না পেলে ICMP "Destination Unreachable" মেসেজ দিয়ে রেসপন্স করে, যা তার নিজস্ব রিসোর্সকে ধীর বা ওভারলোড করে ফেলে।
  • Mitigation:
    • অস্বাভাবিক UDP ট্রাফিক স্পাইকগুলো (spikes) মনিটর করা।
    • রেট লিমিটিং (rate limiting), ফায়ারওয়াল এবং ইন্ট্রুশন প্রিভেনশন সিস্টেম (IPS) ব্যবহার করা।
    • বৃহৎ পরিসরের অ্যাটাকের জন্য ডেডিকেটেড DDoS প্রটেকশন সার্ভিস ডেপ্লয় করা।

How UDP Interacts with IP

অ্যাপ্লিকেশনগুলির মধ্যে যোগাযোগ স্থাপন করতে UDP, ইন্টারনেট প্রোটোকল (IP)-এর ওপরে কাজ করে। এটি কীভাবে ডেটা ট্রান্সমিশন করে তা নিচে ধাপে ধাপে দেওয়া হলো:

  1. অ্যাপ্লিকেশন ডেটার সাথে ডেস্টিনেশনের বিস্তারিত (details) UDP-কে প্রদান করে।
  2. UDP এর নিজস্ব হেডার যুক্ত করে, যার মধ্যে সোর্স পোর্ট, ডেস্টিনেশন পোর্ট, লেংথ এবং চেক্সাম থাকে।
  3. UDP ডেটাগ্রামটি অ্যাড্রেসিং এবং রাউটিংয়ের জন্য IP লেয়ারে পাঠানো হয়।
  4. IP তার নিজস্ব হেডার যুক্ত করে এবং প্যাকেটটি ডেস্টিনেশন হোস্টে ফরোয়ার্ড করে।
  5. রিসিভার বা প্রাপকের প্রান্তে, UDP তার হেডার সরিয়ে ডেটাগুলো টার্গেট অ্যাপ্লিকেশনে ডেলিভার করে।

Differences between TCP and UDP (TCP এবং UDP এর পার্থক্য)

TCP (Transmission Control Protocol) এবং UDP (User Datagram Protocol) হলো Transport Layer-এর অন্যতম গুরুত্বপূর্ণ দুটি প্রোটোকল। উভয়ই এন্ড-টু-এন্ড যোগাযোগ প্রদান করে, তবে এদের কাজ করার ধরন সম্পূর্ণ আলাদা। নিচে এদের মূল পার্থক্যগুলো তুলে ধরা হলো:

TCP (Transmission Control Protocol) UDP (User Datagram Protocol)
Connection-oriented (কানেকশন ওরিয়েন্টেড); ডেটা পাঠানোর আগে ৩-ওয়ে হ্যান্ডশেক ব্যবহার করে। Connectionless (কানেকশনলেস); কোনো সিগন্যাল বা হ্যান্ডশেক ছাড়াই ডেটা পাঠানো শুরু করে।
ডেটা ডেলিভারির Reliable গ্যারান্টি প্রদান করে। ডেটা ডেলিভারির কোনো গ্যারান্টি নেই (প্যাকেট হারাতে পারে)।
Acknowledgment (ACK) ব্যবহার করে ডেটা প্রাপ্তির নিশ্চয়তা দেয়। কোনো Acknowledgment ব্যবহার করে না।
হারিয়ে যাওয়া প্যাকেট (lost packets) পুনরায় পাঠায় (Retransmits)। প্যাকেট হারিয়ে গেলে পুনরায় পাঠানোর (retransmission) সাপোর্ট নেই।
প্যাকেটগুলোর ক্রমানুসারে (In-order) পৌঁছানো নিশ্চিত করে। ডেটার কোনো অর্ডার নিশ্চিত করে না; আউলো-ঝাউলো পৌঁছাতে পারে।
ফ্লো কন্ট্রোল এবং কনজেশন কন্ট্রোল (Flow/Congestion Control) প্রদান করে। কোনো ফ্লো বা কনজেশন কন্ট্রোল নেই।
অতিরিক্ত ওভারহেডের কারণে গতি ধীর (Slower speed)। কম ওভারহেডের কারণে অনেক বেশি দ্রুতগতি সম্পন্ন (Faster speed)।
হেডারের সাইজ পরিবর্তনশীল (Variable: 20-60 bytes)। হেডারের সাইজ নির্দিষ্ট (Fixed size: 8 bytes)।
ব্রডকাস্টিং (Broadcasting) বা মাল্টিকাস্টিং সাপোর্ট করে না। ব্রডকাস্টিং এবং মাল্টিকাস্টিং সাপোর্ট করে।
Application: HTTP, HTTPS, FTP, SMTP-তে ব্যবহৃত হয়। Application: DNS, DHCP, VoIP, ভিডিও স্ট্রিমিং (Video Streaming)-এ ব্যবহৃত হয়।
