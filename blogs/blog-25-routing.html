What is Routing?

March 8, 2026 | Networking

Network Routing হলো একটি বা একাধিক নেটওয়ার্কের মধ্য দিয়ে Data প্যাকেট পাঠানোর সর্বোত্তম পথ নির্বাচন করার প্রক্রিয়া। ইন্টারনেটে Data প্যাকেটগুলো সঠিক গন্তব্যে দক্ষতার সাথে পৌঁছানো নিশ্চিত করতে Routing একটি অত্যন্ত গুরুত্বপূর্ণ ভূমিকা পালন করে।

  • Routing, উৎস থেকে গন্তব্য পর্যন্ত IP প্যাকেট কোন পথে যাবে তা নির্ধারণ করে।
  • এটি ইন্টারনেটের মতো Packet-switched নেটওয়ার্কে ব্যবহৃত হয়।
  • গতি, নির্ভরযোগ্যতা এবং পারফরম্যান্স অপ্টিমাইজ করতে বিভিন্ন Routing পদ্ধতি ব্যবহার করা হয়।

Router কী?

একটি Router হলো একটি নেটওয়ার্কিং ডিভাইস যা গন্তব্য IP অ্যাড্রেস যাচাই করে এবং সেরা পথ বেছে নিয়ে বিভিন্ন নেটওয়ার্কের মধ্যে Data কীভাবে চলাচল করবে তা নিয়ন্ত্রণ করে।

  • নেটওয়ার্কগুলোর মধ্যে Data প্যাকেট ফরোয়ার্ড করে।
  • OSI Model-এর Layer 3 (Network Layer)-এ কাজ করে।
  • Data ট্রান্সফারের জন্য সেরা রুট নির্বাচন করে।

একটি Data প্যাকেটের Routing নিম্নলিখিতভাবে হয়:

  1. Source Node (Sender) Data প্যাকেটটি নেটওয়ার্কে পাঠায় এবং প্যাকেটের Header-এ IP অ্যাড্রেস যুক্ত করে।
  2. নিকটতম Router Data প্যাকেটটি গ্রহণ করে এবং কিছু Metric-এর ভিত্তিতে অন্য Routers-এ পাঠিয়ে দেয়।
  3. Data প্যাকেটটি তার নির্দিষ্ট গন্তব্যে পৌঁছানো পর্যন্ত এই প্রক্রিয়া বারবার ঘটে।

গুরুত্বপূর্ণ তথ্য: একটি প্যাকেট সর্বোচ্চ যতটি Hop করতে পারে তার একটি সীমা (Limit) আছে। এই সীমা অতিক্রম করলে প্যাকেটটিকে Dropped বা Lost হিসেবে গণ্য করা হয়।

Routing-এর বিভিন্ন ধরন (Types of Routing)

Routing সাধারণত ৩ ধরনের, প্রতিটি ভিন্ন উদ্দেশ্যে ব্যবহৃত হয় এবং আলাদা কার্যকারিতা প্রদান করে।

১. Static Routing

Static Routing হলো একটি Non-adaptive (অ-অভিযোজিত) পদ্ধতি যেখানে নেটওয়ার্ক অ্যাডমিনিস্ট্রেটর ম্যানুয়ালি রুট কনফিগার করেন। এটি ছোট নেটওয়ার্কের জন্য সবচেয়ে উপযুক্ত এবং Routing সিদ্ধান্তের উপর সম্পূর্ণ নিয়ন্ত্রণ প্রদান করে।

  • রুটগুলো ম্যানুয়ালি যুক্ত করা হয়।
  • প্যাকেট Routing-এ সম্পূর্ণ নিয়ন্ত্রণ থাকে।
  • সহজ কিন্তু বড় নেটওয়ার্কে পরিচালনা করা কঠিন।

২. Dynamic Routing

Dynamic Routing হলো একটি Automatic ও Adaptive (স্বয়ংক্রিয় ও অভিযোজিত) পদ্ধতি যেখানে Routers অ্যালগরিদম ব্যবহার করে পথ নির্বাচন করে। এর নমনীয়তার কারণে এটি আধুনিক নেটওয়ার্কে ব্যাপকভাবে ব্যবহৃত হয়।

  • রুটগুলো স্বয়ংক্রিয়ভাবে আপডেট হয়।
  • নেটওয়ার্কের পরিবর্তনে নিজেকে মানিয়ে নেয়।
  • বড় নেটওয়ার্কের জন্য উপযুক্ত।

৩. Default Routing

Default Routing-এ যখন কোনো নির্দিষ্ট রুট পাওয়া যায় না, তখন প্যাকেটগুলো একটি পূর্বনির্ধারিত Gateway-তে পাঠানো হয়। এটি সাধারণত একটি মাত্র Exit Point আছে এমন নেটওয়ার্কে ব্যবহৃত হয়。

  • কোনো রুট না পেলে ব্যবহার করা হয়।
  • সহজ এবং দক্ষ সেটআপ।
  • Default রুট: 0.0.0.0/0

Routing-এর কার্যপ্রণালী (Working Principle)

Routing, একটি নেটওয়ার্কে Source Node থেকে Destination Node পর্যন্ত সবচেয়ে ছোট পথ খুঁজে বের করার মাধ্যমে কাজ করে। নিচে ধাপে ধাপে Routing-এর কার্যপ্রণালী দেওয়া হলো:

Step 1: Communication Start — একটি ডিভাইস (Client বা Server) HTTP প্রোটোকল ব্যবহার করে যোগাযোগ শুরু করে।
Step 2: Data Packets — Data ছোট ছোট প্যাকেটে ভাগ করা হয়। প্রতিটি প্যাকেটে গন্তব্য IP অ্যাড্রেস ট্যাগ করা থাকে।
Step 3: Routing Table — Routers একটি Routing Table ব্যবহার করে গন্তব্যে পৌঁছানোর সেরা ও সবচেয়ে ছোট পথ খুঁজে বের করে।
Step 4: Hopping — প্যাকেটগুলো গন্তব্যে পৌঁছানোর জন্য একাধিক Routers (Hops) এর মধ্য দিয়ে ভ্রমণ করে। Hop সংখ্যার একটি সীমা আছে; অতিক্রম করলে প্যাকেট Drop হয় এবং পুনরায় পাঠানো হয়।
Step 5: Destination — প্যাকেটগুলো গন্তব্যে পৌঁছায়, পুনরায় একত্রিত হয় এবং Error চেক করা হয়।

একাধিক পথ থাকলেও সমস্ত উপযুক্ত Metric পূরণ হলে Data প্যাকেটগুলো সবচেয়ে কম Hop Count-এর (সবচেয়ে ছোট) পথে পাঠানো হয়।

Routing-এর এই প্রক্রিয়ায় ৩টি প্রধান উপাদান থাকে:

  • Sender (প্রেরক): যে ডিভাইস থেকে Data পাঠানো হয়।
  • Receiver (প্রাপক): যে ডিভাইস Data গ্রহণ করে।
  • Routers: মধ্যবর্তী ডিভাইসগুলো যা সেরা পথ নির্ণয় করে Data প্যাকেট পাঠায়।

উৎস থেকে গন্তব্যে একাধিক পথ থাকলেও, সমস্ত শর্ত পূরণ হলে সবচেয়ে কম Hop Count-এর পথ (Shortest Path) ব্যবহার করা হয়।

প্রধান Routing Protocols

Protocol ধরন (Type) বিবরণ
RIP Distance-Vector Hop Count ব্যবহার করে সেরা পথ নির্বাচন করে (Routing Information Protocol)।
OSPF Link-State Dijkstra অ্যালগরিদম ব্যবহার করে সবচেয়ে ছোট পথ খোঁজে (Open Shortest Path First)।
EIGRP Hybrid Distance-Vector ও Link-State উভয়ের বৈশিষ্ট্য একত্রিত করে (Enhanced Interior Gateway Routing Protocol)।
BGP Path-Vector ইন্টারনেটে বিভিন্ন Autonomous System-এর মধ্যে Routing-এর জন্য ব্যবহৃত হয় (Border Gateway Protocol)।
IS-IS Link-State ISP-এর মতো বড় নেটওয়ার্কে ব্যবহৃত হয় (Intermediate System to Intermediate System)।

Routing Metrics (পরিমাপ পদ্ধতি)

Routing Protocol-এর উদ্দেশ্য হলো সমস্ত উপলব্ধ পথ সম্পর্কে জানা, Routing Table তৈরি করা এবং নির্দিষ্ট Metric-এর ভিত্তিতে Routing সিদ্ধান্ত নেওয়া। দুটি প্রাথমিক ধরনের Routing Protocol আছে:

১. Distance Vector Routing

এই ধরনের Routing Protocol-এ, নেটওয়ার্কের সকল Node নিয়মিত বিরতিতে তাদের সংলগ্ন নোডগুলোতে (সরাসরি সংযুক্ত) তাদের Routing Table বিজ্ঞাপন করে। প্রতিটি Router নিয়মিত আপডেট পেলেও সব Node-এর একই সঠিক নেটওয়ার্ক দৃশ্য পেতে সময় লাগতে পারে।

  • Fixed Length Subnet ব্যবহার করে, Scaling-এর জন্য উপযুক্ত নয়।
  • ব্যবহৃত Algorithm: Bellman-Ford Algorithm (সবচেয়ে ছোট পথ খোঁজার জন্য)।

২. Link State Routing

Link State Routing হলো একটি Dynamic Routing Protocol যেখানে Routers শুধুমাত্র নেটওয়ার্কে পরিবর্তন হলে আপডেট পাঠায়, যা Bandwidth বাঁচায়। Routers সেরা পথ খুঁজে পেতে Link সম্পর্কিত তথ্য (Cost, Hop Count) শেয়ার করে।

  • আরো ভালো ও Scalable অ্যাড্রেসিংয়ের জন্য VLSM (Variable Length Subnet Mask) ব্যবহার করে।
  • সবচেয়ে ছোট পথ গণনা করতে Dijkstra's Algorithm ব্যবহার করে।
  • Node-গুলোর মধ্যে Cost বিভিন্ন Routing Metrics দ্বারা পরিমাপ করা হয় (যেমন: Hop Count, Delay, Bandwidth)।

Routing Metrics পরিমাপের বিভিন্ন উপায়:

  • Hop Count: একটি Data প্যাকেট গন্তব্যে পৌঁছাতে কতটি Node অতিক্রম করে তার সংখ্যা। কম Hop মানে ছোট এবং দ্রুত পথ।
  • Bandwidth Consumption: Routing এমন পথ বেছে নেয় যা Network Bandwidth দক্ষতার সাথে ব্যবহার করে এবং Congestion এড়ায়।
  • Delay: উৎস থেকে গন্তব্যে Data প্যাকেট পৌঁছাতে কতক্ষণ সময় লাগে। Routing এই Delay কমাতে চেষ্টা করে।
  • Load: একটি নেটওয়ার্ক পথ কতটা ব্যস্ত। দ্রুত Delivery-এর জন্য প্যাকেটগুলো কম ব্যস্ত পথে পাঠানো হয়।
  • Reliability: Routing স্থিতিশীল ও সহজলভ্য পথ বেছে নেয় যাতে Data নিরাপদে গন্তব্যে পৌঁছায়।

সুবিধা ও অসুবিধা (Pros & Cons)

সুবিধাসমূহ (Advantages):

  • দক্ষ Data প্রেরণ: সেরা পথ বেছে নিয়ে দক্ষতার সাথে Data পাঠাতে সাহায্য করে।
  • Automated Routing: Algorithm ব্যবহার করে Manual কাজ কমায়, তবে সরাসরি নিয়ন্ত্রণ কিছুটা কম থাকে।
  • Manual Routing: আরও ভালো নিয়ন্ত্রণ প্রদান করে, যদিও পরিচালনা করতে বেশি প্রচেষ্টা প্রয়োজন।
  • Scalable Routing: বড় নেটওয়ার্কের জন্য ভালোভাবে কাজ করে এবং Traffic ও Data Loss কমাতে Load Balancing সমর্থন করে।

অসুবিধাসমূহ (Disadvantages):

  • Static Routing: শুধুমাত্র ছোট নেটওয়ার্কের জন্য উপযুক্ত; বড় নেটওয়ার্কে পরিচালনা করা কঠিন এবং নিরাপত্তা কম।
  • Dynamic Routing: Manual নিয়ন্ত্রণ কম এবং বেশি Bandwidth ও System Resource ব্যবহার করে।
  • Default Routing: ব্যবহার করা সহজ কিন্তু Default পথটি সঠিকভাবে কনফিগার না হলে সমস্যা হতে পারে।
