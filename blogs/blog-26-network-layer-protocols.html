Network Layer Protocols

March 8, 2026 | Networking

Network Layer একটি নেটওয়ার্কে সংযুক্ত এক Host থেকে অন্য Host-এ Data বা যোগাযোগ প্রেরণের জন্য দায়ী। Data কীভাবে Transfer হয় তা বর্ণনা করার পরিবর্তে, এটি দক্ষ Transmission-এর কৌশল বাস্তবায়ন করে। দক্ষ যোগাযোগ নিশ্চিত করতে Network Layer-এ বিভিন্ন Protocol ব্যবহার করা হয়। Data-কে প্যাকেটে বা অত্যন্ত বড় Data-এর ক্ষেত্রে ছোট Sub-packet-এ ভাগ করা হয়। প্রতিটি Protocol-এর নিজস্ব বৈশিষ্ট্য ও সুবিধা রয়েছে।

Network Layer-এর কাজ (Functions)

কাজ বিবরণ
Logical Addressing নেটওয়ার্কের প্রতিটি ডিভাইসকে অনন্যভাবে সনাক্ত করার জন্য IP অ্যাড্রেসিং স্কিম প্রদান করে। প্রতিটি Header-এ Sender ও Receiver-এর IP অ্যাড্রেস এবং Network ID ও Host ID থাকে।
Host-to-host Delivery প্যাকেটটি Sender থেকে সঠিক Receiver-এ সফলভাবে পৌঁছানো নিশ্চিত করে।
Fragmentation বড় Data-কে ছোট প্যাকেটে ভাগ করে (কারণ প্রতিটি Node-এর Data গ্রহণের নির্দিষ্ট ক্ষমতা আছে)।
Congestion Control নেটওয়ার্কে প্যাকেটের সংখ্যা বেশি হয়ে গেলে Router প্যাকেট রুট করতে পারে না। Network Layer এই Congestion নিয়ন্ত্রণ করে।
Routing & Forwarding Sender থেকে Receiver-এ প্যাকেট পাঠানোর পথ নির্ধারণ করে। সাধারণত সবচেয়ে ছোট পথ বেছে নেওয়া হয় (Path Vector, Distance Vector, Link State Routing ইত্যাদি)।

Network Layer Protocols — সারসংক্ষেপ

Protocol পূর্ণ নাম কাজ
IP Internet Protocol নেটওয়ার্কে প্রতিটি ডিভাইসকে সনাক্ত করে এবং Data প্যাকেট Transfer করে।
ARP Address Resolution Protocol IP Address (Logical) → MAC Address (Physical)-এ রূপান্তর করে।
RARP Reverse Address Resolution Protocol MAC Address (Physical) → IP Address (Logical)-এ রূপান্তর করে।
ICMP Internet Control Message Protocol Error-রিপোর্টিং এবং নেটওয়ার্ক Diagnostic Protocol।
IGMP Internet Group Message Protocol Multicast যোগাযোগের জন্য ব্যবহৃত Protocol।
Protocol 1

IP (Internet Protocol)

Internet Protocol (IP) নেটওয়ার্কের প্রতিটি ডিভাইসকে অনন্যভাবে সনাক্ত করতে এবং এক Node থেকে অন্য Node-এ Data Transfer করতে সাহায্য করে। IP একটি Connectionless Protocol, তাই এটি Data Delivery-এর গ্যারান্টি দেয় না। সফল Delivery নিশ্চিত করতে উচ্চতর Protocol যেমন TCP ব্যবহার করা হয়।

Internet Protocol দুটি ধরনের:

  • IPv4:
    • ৩২-বিটের অ্যাড্রেস স্কিম প্রদান করে।
    • চারটি সংখ্যার Field যা Dot দ্বারা বিভক্ত (যেমন: 192.168.1.1)।
    • DHCP বা ম্যানুয়ালি কনফিগার করা যায়।
    • Authentication বা Encryption সমর্থন করে না, তাই নিরাপত্তা কম।
    • পাঁচটি ক্লাসে বিভক্ত: Class A, B, C, D এবং E।
  • IPv6:
    • ১২৮-বিটের অ্যাড্রেস স্কিম প্রদান করে।
    • আটটি Alphanumeric Field যা Colon দ্বারা বিভক্ত। Hexadecimal-এ প্রকাশ করা হয়।
    • Authentication ও Encryption সমর্থন করে, তাই নিরাপত্তা বেশি।
    • End-to-end Connection Integrity সমর্থন করে।
    • IPv4-এর তুলনায় অনেক বেশি IP Address প্রদান করে।
Protocol 2

ARP (Address Resolution Protocol)

ARP Logical Address (IP Address) থেকে Physical Address (MAC Address)-এ রূপান্তর করতে ব্যবহৃত হয়। অন্য Node-এর সাথে যোগাযোগ করতে গন্তব্য Node-এর MAC Address জানা প্রয়োজন।

কোনো নেটওয়ার্কের Node যদি অন্য Node-এর Physical Address জানতে চায়, তখন একটি ARP Query Packet পাঠানো হয়। এই প্যাকেটে Source Host-এর IP ও MAC Address এবং Destination Host-এর শুধু IP Address থাকে। নেটওয়ার্কের প্রতিটি Node এই প্যাকেট পায়; যার IP মিলে যায় সে তার MAC Address পাঠিয়ে দেয়।

এই প্যাকেট আদান-প্রদান Traffic বাড়ায়, তাই সিস্টেমগুলো ARP Cache বজায় রাখে যেখানে সম্প্রতি প্রাপ্ত IP-to-MAC বাইন্ডিং সংরক্ষিত থাকে।

ARP কীভাবে কাজ করে?

  1. Host নেটওয়ার্কে একটি ARP Inquiry Packet Broadcast করে, যাতে গন্তব্যের IP Address থাকে।
  2. নেটওয়ার্কের সকল Host এই ARP Packet গ্রহণ ও প্রক্রিয়া করে; কিন্তু শুধুমাত্র যার IP মিলে যায় সে তার Physical Address দিয়ে Reply করে।
  3. প্রাপক Host Physical Address Datagram Header ও Cache Memory-তে যুক্ত করে এবং Sender-কে ফেরত পাঠায়।

ARP Entry-র ধরন (Types of ARP Entries)

  • Static Entry: এই ধরনের Entry তখন তৈরি হয় যখন একজন User ARP Command Utility ব্যবহার করে ম্যানুয়ালি IP-to-MAC Address Association যুক্ত করেন।
  • Dynamic Entry: Dynamic Entry স্বয়ংক্রিয়ভাবে তৈরি হয় যখন Sender পুরো নেটওয়ার্কে তার Message Broadcast করে। Dynamic Entry অস্থায়ী এবং নিয়মিত মুছে ফেলা হয়।
Protocol 3

RARP (Reverse Address Resolution Protocol)

RARP ARP-এর বিপরীতভাবে কাজ করে। এটি Physical Address (MAC Address) থেকে Logical Address (IP Address)-এ রূপান্তর করতে ব্যবহৃত হয়। RARP সিস্টেম ও অ্যাপ্লিকেশনগুলোকে DNS বা Router থেকে তাদের নিজস্ব IP Address পাওয়ার সুবিধা দেয়। এই ধরনের Resolution বিভিন্ন কাজে প্রয়োজন, যেমন Reverse DNS Lookup সম্পাদন করা।

RARP নিম্ন স্তরে (Low Level) কাজ করে এবং সরাসরি নেটওয়ার্ক অ্যাড্রেস প্রয়োজন। Server-এর Reply সাধারণত অল্প তথ্য বহন করে এবং ৩২-বিট Internet Address ব্যবহার করে; এটি Ethernet-এর মতো নেটওয়ার্কের পূর্ণ সম্ভাবনা কাজে লাগায় না।

RARP কীভাবে কাজ করে?

  1. RARP, Network Access Layer-এ থেকে নেটওয়ার্কের দুটি স্থানের মধ্যে Data পাঠায়।
  2. নেটওয়ার্কের প্রতিটি User-এর দুটি বিভিন্ন Address থাকে: MAC (Physical) Address এবং IP (Logical) Address। Software IP Address নির্ধারণ করে এবং Hardware MAC Address ডিভাইসে তৈরি করে।
  3. নেটওয়ার্কে সংযুক্ত যেকোনো সাধারণ Computer RARP Server হিসেবে কাজ করতে পারে। তবে এতে সকল MAC Address-এর সাথে সংশ্লিষ্ট IP Address সংরক্ষিত থাকতে হবে।
  4. Client তার Physical Address ও Ethernet Broadcast Address ব্যবহার করে একটি RARP Request পাঠায়। Server এর জবাবে Client-কে তার IP Address প্রদান করে।
Protocol 4

ICMP (Internet Control Message Protocol)

ICMP IP Protocol Suite-এর একটি অংশ। এটি একটি Error Reporting এবং Network Diagnostic Protocol। নেটওয়ার্কে যেকোনো Error হলে তা ICMP-এর মাধ্যমে নির্দিষ্ট Host-এ জানানো হয়। ICMP Protocol বিভিন্ন Error Reporting ও Diagnostic Message ধারণ করে।

ICMP Message-এর ধরন:

  • Error Message: IP প্যাকেট প্রক্রিয়াকরণের সময় Host বা Router যে সমস্যার সম্মুখীন হয় সে সম্পর্কে Error Message জানায়।
  • Query Message: Host একটি Router বা অন্য Host থেকে তথ্য পেতে Query Message ব্যবহার করে।

ICMP-এর প্রধান Error ধরনসমূহ:

  • Time Exceeded: যখন একটি প্যাকেটের TTL (Time To Live) শেষ হয়ে যায় এবং গন্তব্যে পৌঁছানোর আগেই Drop হয়, তখন এই Error পাঠানো হয়।
  • Redirection: যখন একটি Router নির্ধারণ করে যে আরও ভালো রুট আছে এবং Sender-কে সেই পথে পাঠাতে বলে।
  • Source Quench: যখন Router বা Host অতিরিক্ত প্যাকেট পাওয়ার কারণে Sender-কে প্যাকেট পাঠানোর গতি কমাতে বলে।
  • Destination Unreachable: যখন গন্তব্য Host বা Network পৌঁছানো সম্ভব হয় না, তখন এই Error পাঠানো হয়।
  • Parameter Problems: IP প্যাকেটের Header-এ কোনো সমস্যা থাকলে এই Error তৈরি হয়।

ICMP কীভাবে কাজ করে?

  1. ICMP, IP Suite-এর সবচেয়ে গুরুত্বপূর্ণ Protocol-গুলোর একটি। TCP ও UDP-এর বিপরীতে, ICMP একটি Connectionless Protocol, অর্থাৎ Message পাঠাতে গন্তব্য ডিভাইসের সাথে আগে থেকে সংযোগ স্থাপন করতে হয় না।
  2. ICMP ও TCP-এর কাজ করার পদ্ধতি ভিন্ন। TCP একটি Connection-oriented Protocol; Message পাঠানোর আগে TCP Handshake প্রয়োজন। কিন্তু ICMP Connection ছাড়াই কাজ করে।
  3. ICMP প্যাকেট একটি IP Header সহ Datagram আকারে পাঠানো হয়, যাতে ICMP Data থাকে। একটি ICMP Datagram একটি স্বাধীন Data আইটেম বা প্যাকেটের মতো।
Protocol 5

IGMP (Internet Group Message Protocol)

IGMP একটি Multicasting যোগাযোগ Protocol। এটি Message ও Data প্যাকেট Broadcast করার সময় Resources দক্ষতার সাথে ব্যবহার করে। IGMP, TCP/IP-এ ব্যবহৃত একটি Protocol। IP নেটওয়ার্কযুক্ত নেটওয়ার্কের অন্য Host ও Router-গুলো Multicast যোগাযোগের জন্য IGMP ব্যবহার করে।

অনেক নেটওয়ার্কে সমস্ত Node-এ Message পাঠাতে Multicast Router ব্যবহার করা হয়। কিন্তু সবকটিকে Broadcast করা Network Load বাড়ায়। IGMP, Multicast Router-কে Broadcast চলাকালীন Address করে এই সমস্যার সমাধান করে। যেহেতু Multicast যোগাযোগে একাধিক Sender ও Receiver থাকে, তাই IGMP Streaming Media, Web Conference, Games ইত্যাদি অ্যাপ্লিকেশনে ব্যাপকভাবে ব্যবহৃত হয়।

IGMP কীভাবে কাজ করে?

  1. Dynamic Multicasting ও Multicast Group সমর্থনকারী ডিভাইসগুলো IGMP ব্যবহার করতে পারে।
  2. এই ডিভাইসগুলো ব্যবহার করে Host Multicast Group-এ যোগ দিতে বা ছাড়তে পারে। গ্রাহকদের Group-এ যোগ করা বা সরানোও সম্ভব।
  3. Host ও স্থানীয় Multicast Router এই যোগাযোগ Protocol ব্যবহার করে। একটি Multicast Group তৈরি হলে, প্যাকেটের গন্তব্য IP Address পরিবর্তন করে Multicast Group Address দেওয়া হয়, যা Class D IP Address Range-এর মধ্যে পড়ে।
