TCP/IP Model and its Layers

2 January, 2026 | Protocol

TCP/IP Model কী? (What is TCP/IP Model?)

TCP/IP (Transmission Control Protocol/Internet Protocol) Model হলো একটি Layered Networking Framework যা বর্ণনা করে কীভাবে Network-এর মাধ্যমে Device-গুলোর মধ্যে Data আদান-প্রদান করা হয়। এটি Standard Protocol ব্যবহার করে Reliable এবং দক্ষ Data Transmission নিশ্চিত করে।

  • এটি একটি Four-layer Architecture যা Application, Transport, Internet এবং Network Access Layer নিয়ে গঠিত।
  • এটি RFC 1122 দ্বারা Standardized করা হয়েছে।
  • এটি সাত-Layer OSI Model-এর তুলনায় সহজ এবং High ব্যবহারিক।
  • এটি Modern Internet এবং Networking System-এর মূল Structure হিসেবে Task করে।

TCP/IP Model-এর Layers (Layers of TCP/IP Model)

1. Network Access Layer (Link)

1. Application Layer

এটি TCP/IP Model-এর সবচেয়ে উপরের Layer, যা User-এর সবচেয়ে কাছাকাছি থাকে। এখানে Web Browser, Email Client এবং File-sharing Tool-গুলো Network-এর সাথে interact করে। এটি User Software এবং নিচের Layer-গুলোর মধ্যে Interface হিসেবে Task করে, যা নিরবচ্ছিন্ন যোগাযোগ নিশ্চিত করে।

  • User Application ও নিচের Layer-এর মধ্যে Bridge: এটি User Application এবং নিচের Network Layer-এর মধ্যে একটি Bridge হিসেবে Task করে।
  • Protocol Support: এটি HTTP, FTP, SMTP, এবং DNS-এর মতো Protocol support করে।
  • Data Formatting: Information যাতে Sender এবং Receiver উভয় পক্ষই সঠিকভাবে বুঝতে পারে সেভাবে Data Formatting Handle করে।
  • Encryption: Secure যোগাযোগের জন্য Encryption প্রদান করে।
  • Session Management: Running Connection-গুলো Track করতে Session Operation করে।

2. Transport Layer

এটি Device-গুলোর মধ্যে Data-র Reliable এবং দক্ষ Delivery নিশ্চিত করে। এটি Segmentation, Ordering এবং প্রয়োজন অনুযায়ী Retransmission Manage করে।

  • Segmentation এবং Reassembly: বড় Message-কে Packet-এ ভাগ করে এবং Destination এ সেগুলো আবার Pair লাগায়।
  • Reliable Delivery ও Error Handling: TCP Error Test করে, হারানো Data পুনরায় পাঠায় এবং সঠিক ক্রম নিশ্চিত করে।
  • Fast যোগাযোগ: UDP কম-Latency এবং Connectionless Delivery প্রদান করে (Error Test ছাড়াই)।
  • Flow Control: Data প্রবাহ Control করে Receiver অতিরিক্ত Data-র চাপ থেকে রক্ষা করে।
  • Multiplexing: Port number ব্যবহার করে Multiple Application-কে একসাথে Network Share করার সুযোগ দেয়।

TCP (Transmission Control Protocol)

যখন Reliable তা এবং Accurate তা সবচেয়ে High গুরুত্বপূর্ণ তখন TCP ব্যবহৃত হয়। এটি নিশ্চিত করে যে Data ঠিক যেভাবে পাঠানো হয়েছে সেভাবেই পৌঁছাবে।

  • Error Checking: Checksum ব্যবহার করে Data-র Error শনাক্ত করে অখণ্ডতা নিশ্চিত করে।
  • Retransmission: Transmission-এর সময় কোনো Data হারিয়ে গেলে বা নষ্ট হলে TCP স্বয়ংক্রিয়ভাবে তা পুনরায় পাঠায়।
  • Ordered Delivery: Data Packet-এ বিভক্ত হয় এবং TCP নিশ্চিত করে যে এই Packet-গুলো সঠিক ক্র মেপৌঁছাবে।
  • Connection-oriented: Data পাঠানোর আগে Sender ও Receiver মধ্যে Connection তৈরি করে Session বজায় রাখে।
  • ব্যবহার: Web page Load করা, ফাইল Download, Email পাঠানো বা যেখানে Data-র Accurate তা প্রয়োজন।

UDP (User Datagram Protocol)

যখন Accurate তার চেয়ে Speed High গুরুত্বপূর্ণ তখন UDP ব্যবহৃত হয়। এটি Fast Task করে কিন্তু Reliable Delivery-র গ্যারান্টি দেয় না।

  • Error Checking নেই: Data-তে Error আছে কিনা তা যাচাই করে না।
  • Retransmission নেই: হারানো বা ক্ষতিগ্রস্ত Data পুনরায় পাঠায় না।
  • Ordered Delivery নেই: Packet-গুলো যেকোনো ক্র মেপৌঁছাতে পারে।
  • Lightweight: অতিরিক্ত Check না থাকায় এটি Fast এবং কম Resource ব্যবহার করে।
  • ব্যবহার: Live Video Streaming, অনলাইন গেমিং, VoIP কল।

3. Internet Layer

এটি Data Packet-গুলোর Addressing, Packaging এবং Routing করার জন্য দায়ী যাতে সেগুলো Network-এর মাধ্যমে সঠিক Destination এ পৌঁছাতে পারে।

  • লজিক্যাল Address িং: সোর্স এবং Destination Device শনাক্ত করতে IP Address অ্যাসাইন করে।
  • Packet-রাউটিং: Data ভ্রমণের জন্য সেরা Path নির্ধারণ করে।
  • ফ্র্যাগ মেন্টেশন এবং রিঅ্যাসেম্বলি: বড় Packet কে ছোট ছোট ভাগে ভাগ করে এবং Destination এ আবার জোড়া লাগায়।
  • Protocol সাপোর্ট: মূলত IP (Internet Protocol) ব্যবহার করে, সাথে ICMP (এরর রিপোর্টিং) এবং ARP (Address-রেজোলিউশন) ব্যবহার করে।

4. Network Access Layer (Link Layer)

এটি Network Hardware-এর (Cabling, Switch, Wireless) মাধ্যমে Data Physically Transmit করার জন্য দায়ী।

  • Physical Transmission: Ethernet Cable, Fiber Optics বা Wi-Fi-এর মাধ্যমে Raw Bits পাঠায় ও গ্রহণ করে।
  • Framing: Data-কে Frames-এ সাজায় যাতে Device-গুলো তা চিনতে পারে।
  • Error Detection: Checksum বা CRC ব্যবহার করে Transmission Error শনাক্ত করে।
  • MAC Addressing: একই Network segment-এর মধ্যে Device শনাক্ত করতে Hardware Address ব্যবহার করে।
  • Access Control: Multiple Device কীভাবে Physical Medium Share করবে তা নির্ধারণ করে (Collision এড়ানো)।

TCP/IP Model-এর Mechanism (Working of TCP/IP Model)

1. Data পাঠানোর সময় (From Sender to Receiver):

  • Application Layer: User-এর Software Data তৈরি করে নিচের Layer-এ পাঠায়।
  • Transport Layer: Data-কে ছোট ছোট ভাগে (Segments) ভাগ করে এবং Control information (TCP/UDP) যোগ করে।
  • Internet Layer: প্রতিটি Segment-এর সাথে IP Address যোগ করে Packet-এ রূপান্তর করা হয় যাতে Network-এর মাধ্যমে Route করা যায়।
  • Network Access Layer: Packet-গুলোকে Frame-এ রূপান্তর করে Physical Medium-এর (Ethernet, Wi-Fi) মাধ্যমে পাঠানো হয়।

2. Data গ্রহণের সময় (At the Destination):

  • Network Access Layer: Physical Medium থেকে Frame-গুলো গ্রহণ করে এবং Error Check করে।
  • Internet Layer: Frame থেকে Packet বের করে এবং IP Address Check করে সঠিক Device-এ পৌঁছানো নিশ্চিত করে।
  • Transport Layer: Segment-গুলোকে আবার Pair লাগিয়ে মূল Message তৈরি করে (যদি TCP ব্যবহার হয় তবে Error সংশোধন করে)।
  • Application Layer: সম্পূর্ণ Data User Application-এ (যেমন Browser বা Email Client) পৌঁছে দেয়।

Advantage ও সীমাবদ্ধতা

Advantage (Advantages)

  • ব্যাপক ব্যবহার: এটি Internet এবং অধিকাংশ Modern Network-এর ভিত্তি।
  • Platform Independent: বিভিন্ন Hardware এবং Operating System-এ Task করে।
  • Reliable যোগাযোগ: TCP Error checking এবং Delivery Confirmation নিশ্চিত করে।
  • Scalable: ছোট Network থেকে Global Internet scale পর্যন্ত Support করে।
  • Open Standard: এটি বিনামূল্যে ব্যবহারযোগ্য এবং কোনো Single প্রতিষ্ঠানের দ্বারা নিয়ন্ত্রিত নয়।

সীমাবদ্ধতা (Limitations)

  • নতুনদের জন্য জটিল: সব Protocol পুরোপুরি বোঝা কঠিন হতে পারে।
  • Layer-এর কঠোর Rule নেই: OSI-এর মতো Layer-এর সীমানা কঠোর নয়, যা implementation-এ ভিন্নতা আনতে পারে।
  • Overhead: TCP-এর Error checking ফিচারের কারণ ে অতিরিক্ত Data Overhead হতে পারে।
  • Security-র সীমাবদ্ধতা: এটি প্রাথমিকভাবে built-in security দিয়ে Design করা হয়নি (TLS/SSL প্রয়োজন)।
  • Multimedia support কম: Real-time audio/video-র জন্য এটি optimized নয় (অতিরিক্ত Protocol লাগে)।

কেন OSI Model-এর চেয়ে TCP/IP বেশি ব্যবহৃত হয়?

TCP/IP Internet-এ High Popular কারণ এটি সহজ, ব্যবহারিক এবং বাস্তবসম্মত। এর প্রধান কারণ গুলো হলো:

  • সহজ Structure: OSI-এর 7টি Layer-এর তুলনায় TCP/IP-এর মাত্র 4টি Layer, যা implement করা সহজ।
  • Protocol-চালিত Design: TCP/IP তৈরি হয়েছে বাস্তব Protocol-এর ওপর ভিত্তি করে, যেখানে OSI একটি Theoretical Model।
  • Flexible তা ও দৃঢ়তা: এটি বিভিন্ন Hardware-এর সাথে খাপ খায় এবং এর শক্তিশালী Error handling ও Routing Capacity রয়েছে।
  • Open Standard: এটি Open এবং Free হওয়ার কারণ ে সর্বজনীন গ্রহণযোগ্যতা পেয়েছে।
  • বাস্তব ব্যবহার: OSI শেখার জন্য ভালো হলেও, বাস্তব বিশ্বে Internet-এর জন্য TCP/IP-ই ব্যবহৃত হয়।
