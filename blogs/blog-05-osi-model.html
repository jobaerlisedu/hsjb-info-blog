OSI Model and its Layers

2 January, 2026 | Protocol

OSI Model What is? (What is OSI Model?)

OSI (Open Systems Interconnection) Model হলো একটি Theoretical Framework যা বর্ণনা করে কীভাবে ভিন্ন ভিন্ন Computer System একটি Network-এর মাধ্যMay একে অপরের সাথে যোগাযোগ করে। এটি International Organization for Standardization (ISO) দ্বারা ১৯৮৪ সালে Creation করা হয়েছে। এটি Modern Networking এবং Communication System-এর ভিত্তি হিসেবে Task করে এবং Troubleshooting ও Network Design-এ অত্যন্ত সহায়ক।

OSI Model-এর ৭টি Layer (7 Layers of OSI Model)

OSI Model সাতটি পৃথক Layerে বিভক্ত, যেখানে প্রতিটি Layer নির্দিষ্ট Task ও দায়িত্ব রয়েছে। এই Layer নিচে থেকে উপরে সাজানো থাকে:

1. Physical Layer

এটি OSI Model সর্বনিম্ন Layer। এর প্রধান Task হলো Deviceের মধ্যে প্রকৃত ফিজিক্যাল Connection Creation করা এবং Bit Sizeে (0s and 1s) Data ট্রান্সমিট করা।

  • Bit Synchronization: এটি Sender এবং Receiver মধ্যে Bit লেভেলে Synchronization নিশ্চিত করে।
  • Bit রেট Control: প্রতি সেকেন্ডে কতগুলো Bit পাঠানো হবে তা Determination করে (Transmission Rate)।
  • ফিজিক্যাল Topology: Networkে Deviceগুলো কীভাবে যুক্ত থাকবে (Bus, Star, Mesh) তা ঠিক করে।
  • Transmission মোড: Data প্রবাহের Type Determination করে (Simplex, Half-Duplex, Full-Duplex)।
  • Device ও Interface: Hub (Hub), রিপিটার (Repeater), ডি-Connector এবং Cableগুলো এই Layerে Task করে।

2. Data Link Layer

এই Layerটি Node-টু-Node Mayসেজ Deliveryর জন্য দায়ী। এটি ফিজিক্যাল Layer উপর ভিত্তি করে Task করে এবং Data Transferকে ত্রুTeamুক্ত করার চেষ্টা করে।

  • Frameিং (Framing): Network Layer থেকে আসা Packetগুলোকে ছোট ছোট 'Frame'-এ রূপান্তর করে।
  • ফিজিক্যাল Addressিং (MAC Address): প্রতিটি Deviceের হার্ডওয়্যারে থাকা ইউনিক MAC Address ব্যবহার করে সঠিক Deviceে Data পৌঁছানো নিশ্চিত করে।
  • এরর Control: Checkসাম বা CRC ব্যবহার করে Data Transmissionে কোনো Error হয়েছে কিনা তা শনাক্ত করে।
  • ফ্লো Control: Sender যদি খুব Fast Data পাঠায় এবং Receiver তা গ্রহণ করতে না পারে, তবে এটি Data প্রবাহ Control করে।
  • অ্যাক্সেস Control: যখন Multiple Device একই Channel শেয়ার করে, তখন কে আগে Data পাঠাবে তা Determination করে।
  • Device: Switch (Switch) এবং Bridge (Bridge) এই Layerে Task করে।

3. Network Layer

বিভিন্ন Networkে থাকা হোস্টগুলোর মধ্যে Data Packetের Reliable Transfer নিশ্চিত করে এই Layer। এটি সবচেয়ে উপযুক্ত Path (Routing) খুঁজে বের করার জন্য দায়ী।

  • রাউটিং (Routing): Network Layer Protocol সোর্স থেকে Destinationে Data পৌঁছানোর সেরা Path Determination করে।
  • লজিক্যাল Addressিং (IP Address): প্রতিটি Packetকে IP Address মাধ্যMay Symbolিত করে যাতে সেগুলো Networkের যে কোনো প্রান্তে পৌঁছাতে পারে।
  • ইন্টারNetworking: ভিন্ন ভিন্ন Typeের Networkের মধ্যে সংযোগ স্থাপনে সহায়তা করে।
  • Device: Router (Router) এই Layer প্রধান Device।

4. Transport Layer

এটি এন্ড-টু-এন্ড (End-to-End) Mayসেজ Delivery নিশ্চিত করে। এটি ওপরের Layer থেকে আসা Dataকে ছোট অংশে ভাগ করে এবং নিচের Layerে পাঠায়।

  • সেগMayন্টেশন এবং রি-অ্যাসেম্বলি: Mayসেজকে ছোট 'সেগMayন্ট'-এ ভাগ করে এবং Receiver কাছে গেলে আবার সেগুলো যোগ করে মূল Mayসেজ Creation করে।
  • সার্ভিস Point Addressিং (Port Number): Transport Layerে পোর্ট Number ব্যবহার করা হয় যাতে একটি Deviceে Multiple Application চললে সঠিক Applicationে Data পৌঁছায়।
  • Connection Control: এটি Connectionলেস (UDP) বা Connection-ওরিয়েন্টেড (TCP) হতে পারে।
  • Reliable Delivery: TCP Protocol ব্যবহার করে একনলেজMayন্টের মাধ্যMay Dataর সঠিক প্রাপ্যতা নিশ্চিত করে।

5. Session Layer

এটি দুটি Deviceের মধ্যে কমিউনিকেশন সেশন বা কথোপকথন শুরু করা, Operation করা এবং শেষ করার দায়িত্ব পালন করে।

  • সেশন ম্যানেজMayন্ট: Connection Creation, বজায় রাখা এবং প্রয়োজন শেষে বিচ্ছিন্ন করা।
  • Synchronization ও CheckPoint: বড় ফাইল Transferের সময় মাঝে মাঝে CheckPoint ব্যবহার করা হয়, যাতে কোনো Reasonে Network বিচ্ছিন্ন হলে সেখান থেকে আবার শুরু করা যায়।
  • Dialogue Controlার: এটি দুই Deviceের মধ্যে হাফ-Duplex বা ফুল-Duplex মোডে কথা বলা Control করে।

6. Presentation Layer

একে 'Switchবোর্ড' বা 'Translation Layer'ও বলা হয়। এটি Application Layer থেকে আসা Dataকে Networkের উপযোগী ফরম্যাটে রূপান্তর করে।

  • Translation (Translation): ডিফারেন্ট Computer সিস্টেMay (যেমন ASCII বা EBCDIC) Data বোঝার উপযোগী করা।
  • Encryption ও Decryption (Encryption/Decryption): Privacy রক্ষার জন্য Dataকে কোড যুক্ত করা এবং পরে তা খোলা।
  • কম্প্রেশন (Compression): Dataর সাইজ ছোট করা যাতে Fast Networkের মাধ্যMay পাঠানো যায়।

7. Application Layer

এটি OSI Model সর্বোচ্চ Layer, যা সরাসরি ইউজারের সাথে ইন্টারঅ্যাক্ট করে। Web ব্রাউজার, Email Client বা অন্যান্য Network অ্যাপ এই Layerে Task করে।

  • Network ভার্চুয়াল Terminal: এটি রিমোট হোস্টের সাথে সংযোগ স্থাপন করতে সাহায্য করে।
  • ফাইল Transfer, এক্সেস ও ম্যানেজMayন্ট (FTAM): ফাইল আদান-প্রদান ও ম্যানেজ করা।
  • Mayইল সার্ভিস: Email পাঠানো এবং ফরওয়ার্ড করা।
  • Directory সার্ভিস: Global ইনফরMayশন ও বিভিন্ন রিসোর্সের Information প্রদান।

OSI Model-এ Data Flow (Data Flow in OSI Model)

যখন একজন User Data পাঠায় তখন তা Application Layer থেকে শুরু হয়ে নিচের Directionে প্রতিটি Layer-এ কিছু অতিরিক্ত Information (Header/Trailer) যোগ করে Physical Layer পর্যন্ত যায়। একে বলা হয় Encapsulation।

আবার যখন Receiver সেই Data গ্রহণ করে, তখন তা Physical Layer থেকে শুরু হয়ে ওপরের Directionে প্রতিটি Layer-এ সেই অতিরিক্ত Informationগুলো বাদ দিয়ে Application Layer-এ পৌঁছায়। একে বলা হয় Decapsulation।

প্রতিটি Layer-এর Protocol-সমূহ (Protocols per Layer)

  • Application: HTTP, FTP, SMTP, DNS
  • Presentation: TLS, SSL, MIME
  • Session: NetBIOS, RPC, PPTP
  • Transport: TCP, UDP
  • Network: IP, ICMP, IGMP
  • Data Link: Ethernet, PPP, HDLC
  • Physical: USB, Ethernet (Physical layer specs), DSL

কেন OSI Model গুরুত্বপূর্ণ? (Why OSI Model is Important?)

  • 1. Standardization: এটি Networking Hardware ও Software Creationর জন্য একটি Universal Model।
  • 2. Troubleshooting: Network-এ সমস্যা হলে Layer by Layer সেটি শনাক্ত করা সহজ হয়।
  • 3. Layer Independence: এক Layer-এর Change অন্য Layer-কে প্রভাবিত করে না, ফলে নতুন প্রযুক্তি যোগ করা সহজ।
  • 4. Learning Tool: Networking কীভাবে Task করে তা বোঝার জন্য এটি সেরা Academic Model।
