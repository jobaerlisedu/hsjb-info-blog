OSI Model and its Layers

2 January, 2026 | Protocol

OSI Model কী? (What is OSI Model?)

OSI (Open Systems Interconnection) Model হলো একটি Theoretical Framework যা বর্ণনা করে কীভাবে ভিন্ন ভিন্ন Computer System একটি Network-এর মাধ্যমে একে অপরের সাথে যোগাযোগ করে। এটি International Organization for Standardization (ISO) দ্বারা 1984 সালে তৈরি করা হয়েছে। এটি Modern Networking এবং Communication System-এর ভিত্তি হিসেবে Task করে এবং Troubleshooting ও Network Design-এ অত্যন্ত সহায়ক।

OSI Model-এর 7টি Layer (7 Layers of OSI Model)

OSI Model সাতটি পৃথক Layer ে বিভক্ত, যেখানে প্রতিটি Layer নির্দিষ্ট Task ও দায়িত্ব রয়েছে। এই Layer নিচে থেকে উপরে সাজানো থাকে:

7. Application Layer
6. Presentation Layer
5. Session Layer
4. Transport Layer
3. Network Layer
2. Data Link Layer
1. Physical Layer

1. Physical Layer

এটি OSI Model সর্বনিম্ন Layer। এর প্রধান Task হলো Device-এর মধ্যে প্রকৃত ফিজিক্যাল Connection তৈরি করা এবং Bit Size ে (0s and 1s) Data ট্রান্সমিট করা।

  • Bit Synchronization: এটি Sender এবং Receiver মধ্যে Bit লেভেলে Synchronization নিশ্চিত করে।
  • Bit-রেট Control: প্রতি সেকেন্ডে কতগুলো Bit পাঠানো হবে তা নির্ধারণ করে (Transmission Rate)।
  • ফিজিক্যাল Topology: Network ে Device-গুলো কীভাবে যুক্ত থাকবে (Bus, Star, Mesh) তা ঠিক করে।
  • Transmission মোড: Data প্রবাহের Type নির্ধারণ করে (Simplex, Half-Duplex, Full-Duplex)।
  • Device ও Interface: Hub (Hub), রিপিটার (Repeater), ডি-Connector এবং Cable-গুলো এই Layer ে Task করে।

2. Data Link Layer

এই Layer টি Node-টু-Node মেসেজ Delivery-র জন্য দায়ী। এটি ফিজিক্যাল Layer উপর ভিত্তি করে Task করে এবং Data Transfer কে ত্রুটিমুক্ত করার চেষ্টা করে।

  • Frame িং (Framing): Network Layer থেকে আসা Packet-গুলোকে ছোট ছোট 'Frame'-এ রূপান্তর করে।
  • ফিজিক্যাল Address িং (MAC Address): প্রতিটি Device-এর হার্ডওয়্যারে থাকা ইউনিক MAC Address ব্যবহার করে সঠিক Device ে Data পৌঁছানো নিশ্চিত করে।
  • এরর Control: Check সাম বা CRC ব্যবহার করে Data Transmission ে কোনো Error হয়েছে কিনা তা শনাক্ত করে।
  • ফ্লো Control: Sender যদি খুব Fast Data পাঠায় এবং Receiver তা গ্রহণ করতে না পারে, তবে এটি Data প্রবাহ Control করে।
  • অ্যাক্সেস Control: যখন Multiple Device একই Channel শেয়ার করে, তখন কে আগে Data পাঠাবে তা নির্ধারণ করে।
  • Device: Switch (Switch) এবং Bridge (Bridge) এই Layer ে Task করে।

3. Network Layer

বিভিন্ন Network ে থাকা হোস্টগুলোর মধ্যে Data Packet-এর Reliable Transfer নিশ্চিত করে এই Layer। এটি সবচেয়ে উপযুক্ত Path (Routing) খুঁজে বের করার জন্য দায়ী।

  • রাউটিং (Routing): Network Layer Protocol সোর্স থেকে Destination এ Data পৌঁছানোর সেরা Path নির্ধারণ করে।
  • লজিক্যাল Address িং (IP Address): প্রতিটি Packet কে IP Address মাধ্যমে Symbol িত করে যাতে সেগুলো Network-এর যে কোনো প্রান্তে পৌঁছাতে পারে।
  • ইন্টার Networking: ভিন্ন ভিন্ন Type-এর Network-এর মধ্যে সংযোগ স্থাপনে সহায়তা করে।
  • Device: Router (Router) এই Layer প্রধান Device।

4. Transport Layer

এটি এন্ড-টু-এন্ড (End-to-End) মেসেজ Delivery নিশ্চিত করে। এটি ওপরের Layer থেকে আসা Data কে ছোট অংশে ভাগ করে এবং নিচের Layer ে পাঠায়।

  • সেগমেন্টেশন এবং রি-অ্যাসেম্বলি: মেসেজকে ছোট 'সেগমেন্ট'-এ ভাগ করে এবং Receiver কাছে গেলে আবার সেগুলো যোগ করে মূল মেসেজ তৈরি করে।
  • সার্ভিস Point Address িং (Port Number): Transport Layer ে পোর্ট Number ব্যবহার করা হয় যাতে একটি Device ে Multiple Application চললে সঠিক Application ে Data পৌঁছায়।
  • Connection Control: এটি Connection লেস (UDP) বা Connection-ওরিয়েন্টেড (TCP) হতে পারে।
  • Reliable Delivery: TCP Protocol ব্যবহার করে একনলেজমেন্টের মাধ্যমে Data-র সঠিক প্রাপ্যতা নিশ্চিত করে।

5. Session Layer

এটি দুটি Device-এর মধ্যে কমিউনিকেশন সেশন বা কথোপকথন শুরু করা, Operation করা এবং শেষ করার দায়িত্ব পালন করে।

  • সেশন ম্যানেজমেন্ট: Connection তৈরি, বজায় রাখা এবং প্রয়োজন শেষে বিচ্ছিন্ন করা।
  • Synchronization ও CheckPoint: বড় ফাইল Transfer-এর সময় মাঝে মাঝে CheckPoint ব্যবহার করা হয়, যাতে কোনো কারণ ে Network বিচ্ছিন্ন হলে সেখান থেকে আবার শুরু করা যায়।
  • Dialogue Control ার: এটি দুই Device-এর মধ্যে হাফ-Duplex বা ফুল-Duplex মোডে কথা বলা Control করে।

6. Presentation Layer

একে 'Switch বোর্ড' বা 'Translation Layer'ও বলা হয়। এটি Application Layer থেকে আসা Data কে Network-এর উপযোগী ফরম্যাটে রূপান্তর করে।

  • অনুবাদ (Translation): ডিফারেন্ট Computer সিস্টেমে (যেমন ASCII বা EBCDIC) Data বোঝার উপযোগী করা।
  • Encryption ও Decryption (Encryption/Decryption): গোপনীয়তা রক্ষার জন্য Data কে কোড যুক্ত করা এবং পরে তা খোলা।
  • কম্প্রেশন (Compression): Data-র সাইজ ছোট করা যাতে Fast Network-এর মাধ্যমে পাঠানো যায়।

7. Application Layer

এটি OSI Model সর্বোচ্চ Layer, যা সরাসরি ইউজারের সাথে ইন্টারঅ্যাক্ট করে। Web ব্রাউজার, Email Client বা অন্যান্য Network অ্যাপ এই Layer ে Task করে।

  • Network ভার্চুয়াল Terminal: এটি রিমোট হোস্টের সাথে সংযোগ স্থাপন করতে সাহায্য করে।
  • ফাইল Transfer, এক্সেস ও ম্যানেজমেন্ট (FTAM): ফাইল আদান-প্রদান ও ম্যানেজ করা।
  • মেইল সার্ভিস: Email পাঠানো এবং ফরওয়ার্ড করা।
  • Directory সার্ভিস: Global ইনফরমেশন ও বিভিন্ন রিসোর্সের Information প্রদান।

OSI Model-এ Data Flow (Data Flow in OSI Model)

যখন একজন User Data পাঠায় তখন তা Application Layer থেকে শুরু হয়ে নিচের দিক ে প্রতিটি Layer-এ কিছু অতিরিক্ত Information (Header/Trailer) যোগ করে Physical Layer পর্যন্ত যায়। একে বলা হয় Encapsulation।

আবার যখন Receiver সেই Data গ্রহণ করে, তখন তা Physical Layer থেকে শুরু হয়ে ওপরের দিক ে প্রতিটি Layer-এ সেই অতিরিক্ত Information-গুলো বাদ দিয়ে Application Layer-এ পৌঁছায়। একে বলা হয় Decapsulation।

প্রতিটি Layer-এর Protocol-সমূহ (Protocols per Layer)

  • Application: HTTP, FTP, SMTP, DNS
  • Presentation: TLS, SSL, MIME
  • Session: NetBIOS, RPC, PPTP
  • Transport: TCP, UDP
  • Network: IP, ICMP, IGMP
  • Data Link: Ethernet, PPP, HDLC
  • Physical: USB, Ethernet (Physical layer specs), DSL

কেন OSI Model গুরুত্বপূর্ণ? (Why OSI Model is Important?)

  • 1. Standardization: এটি Networking Hardware ও Software তৈরি র জন্য একটি Universal Model।
  • 2. Troubleshooting: Network-এ সমস্যা হলে Layer by Layer সেটি শনাক্ত করা সহজ হয়।
  • 3. Layer Independence: এক Layer-এর Change অন্য Layer-কে প্রভাবিত করে না, ফলে নতুন প্রযুক্তি যোগ করা সহজ।
  • 4. Learning Tool: Networking কীভাবে Task করে তা বোঝার জন্য এটি সেরা Academic Model।
